### 什么时候需要 Content Provider？

"在开始写代码之前，"黛琳举起白板笔，"我们先聊聊——什么时候真的需要 Content Provider？"

洛芙想了想："把数据分享给其他 app？"

```
什么时候需要 Content Provider？
├── 向其他应用提供复杂数据或文件
```

### 第一步：设计数据存储

"Provider 本身只是一个'接口层'，"黛琳在白板上写下几个选项，"它不存数据，它只是负责接收请求、调用数据库、返回结果。那——我们用什么存？"

洛芙掰着手指头数："SQLite？Room？文件？"

"都可以，"希尔说，"官方文档说得很清楚："

| 存储类型 | 适用场景 |
|---------|---------|
| SQLite/Room | 结构化数据（表格、行、列） |
| 文件 | 图片、音频、视频等媒体文件 |
| 网络 | 云端数据（同步到本地再提供）|

"最简单的是 SQLite，"希尔说，"代码最直观。我们今天用 SQLite 吧。"

黛琳点头，开始写数据库 helper 类：

```kotlin
// DiaryDbHelper：管理 SQLite 数据库的创建和版本
// 官方推荐：如果是结构化数据，使用 SQLite 或 Room
class DiaryDbHelper(context: Context) : SQLiteOpenHelper(
    context,
    DATABASE_NAME,
    null,
    DATABASE_VERSION
) {

    companion object {
        const val DATABASE_NAME = "diary.db"
        const val DATABASE_VERSION = 1

        // 建表语句
        // 官方提醒：表数据必须有一个"主键"列
        // 推荐使用 _id，因为 CursorAdapter 需要这个列名
        const val SQL_CREATE_DIARY = """
            CREATE TABLE diary (
                _id INTEGER PRIMARY KEY AUTOINCREMENT,
                title TEXT NOT NULL,
                content TEXT NOT NULL,
                mood TEXT,
                weather TEXT,
                location TEXT,
                created_at INTEGER NOT NULL,
                updated_at INTEGER NOT NULL
            )
        """
    }

    override fun onCreate(db: SQLiteDatabase) {
        // 第一次创建数据库时调用
        db.execSQL(SQL_CREATE_DIARY)
    }

    override fun onUpgrade(db: SQLiteDatabase, oldVersion: Int, newVersion: Int) {
        // 数据库升级时调用
        // 官方提醒：如果变更不兼容，要增加版本号
        // 简单粗暴：删掉重建（生产环境要用 Migration）
        db.execSQL("DROP TABLE IF EXISTS diary")
        onCreate(db)
    }
}
```

"这就是地基，"希尔说，"有了这个，我们才能往里面存日记。"

### 第二步：设计 Content URI

"在写 Provider 之前，我们先设计 URI。"黛琳翻到白板新的一页，"这是最核心的东西——Provider 怎么知道调用者想要什么数据？靠 URI。"

```kotlin
// Content URI 结构
// content://<authority>/<path>/<id>

// 实际例子：
content://com.campapp.provider/diary        // 所有日记列表
content://com.campapp.provider/diary/5      // ID = 5 的那篇日记
content://com.campapp.provider/diary/5/tag  // 那篇日记的所有标签
```

"Authority 是什么？"洛芙问。

"就是 Provider 的'店名'，"黛琳解释，"官方推荐使用反转域名来避免冲突："

```kotlin
// Authority 设计
// 官方推荐：使用反转域名作为 authority
// 例如：包名是 com.example.app
// authority 就是 com.example.app.provider
const val AUTHORITY = "com.campapp.provider"
```

"Path 就是表名，"希尔补充，"ID 就是具体哪一行。"

---

### 通配符：# 和 *

"URI 匹配时可以用通配符，"黛琳画出示意图：

| 通配符 | 含义 | 例子 |
|--------|------|------|
| `#` | 匹配任意数字 | `diary/#` 匹配 `diary/1`, `diary/123` |
| `*` | 匹配任意字符串 | `diary/*` 匹配任意 path |

```kotlin
// UriMatcher：URI 路由器
// 官方推荐：使用 UriMatcher 把 URI 映射到整数代码
val uriMatcher = UriMatcher(UriMatcher.NO_MATCH).apply {
    // addURI(authority, path, matchCode)
    addURI("com.campapp.provider", "diary", DIARIES)      // 1
    addURI("com.campapp.provider", "diary/#", DIARY_ID)   // 2
    addURI("com.campapp.provider", "diary/#/tag", DIARY_TAGS) // 3
}
```

### 第三步：实现 ContentProvider 的六个核心方法

洛芙把键盘拉到自己面前，兴奋地开始敲代码：

```kotlin
class DiaryProvider : ContentProvider() {

    private lateinit var dbHelper: DiaryDbHelper

    // 1. onCreate：Provider 启动时调用
    // 官方提醒：只做快速初始化，耗时操作延迟到真正需要时
    override fun onCreate(): Boolean {
        dbHelper = DiaryDbHelper(context!!)
        return true
    }

    // 2. query：查询数据
    // 官方提醒：必须返回 Cursor，如果没有数据返回 count=0 的 Cursor，不是 null
    override fun query(
        uri: Uri,
        projection: Array<out String>?,  // 要查询哪些列
        selection: String?,              // WHERE 条件
        selectionArgs: Array<out String>?,  // WHERE 参数
        sortOrder: String?               // 排序
    ): Cursor? {
        val db = dbHelper.readableDatabase
        val cursor = when (uriMatcher.match(uri)) {
            DIARIES -> db.query(DiaryContract.Diary.TABLE_NAME, projection,
                selection, selectionArgs, null, null, sortOrder)
            DIARY_ID -> {
                val id = ContentUris.parseId(uri)
                db.query(DiaryContract.Diary.TABLE_NAME, projection,
                    "${DiaryContract.Diary.COLUMN_ID} = ?",
                    arrayOf(id.toString()), null, null, null)
            }
            else -> throw IllegalArgumentException("Unknown URI: $uri")
        }
        // 关键：设置通知 URI，这样数据变化时观察者能收到通知
        cursor.setNotificationUri(context?.contentResolver, uri)
        return cursor
    }

    // 3. insert：插入数据
    // 官方提醒：返回新行的 URI（用 ContentUris.withAppendedId 拼接）
    override fun insert(uri: Uri, values: ContentValues?): Uri? {
        if (uriMatcher.match(uri) != DIARIES) {
            throw IllegalArgumentException("Invalid URI for insert: $uri")
        }
        val db = dbHelper.writableDatabase
        val now = System.currentTimeMillis()
        values?.put(DiaryContract.Diary.COLUMN_CREATED_AT, now)
        values?.put(DiaryContract.Diary.COLUMN_UPDATED_AT, now)
        val id = db.insert(DiaryContract.Diary.TABLE_NAME, null, values)
        if (id > 0) {
            val newUri = ContentUris.withAppendedId(DiaryContract.Diary.CONTENT_URI, id)
            context?.contentResolver?.notifyChange(newUri, null)
            return newUri
        }
        return null
    }

    // 4. update：更新数据
    // 官方提醒：返回影响行数
    override fun update(uri: Uri, values: ContentValues?, selection: String?,
                       selectionArgs: Array<out String>?): Int {
        val db = dbHelper.writableDatabase
        values?.put(DiaryContract.Diary.COLUMN_UPDATED_AT, System.currentTimeMillis())
        val rows = when (uriMatcher.match(uri)) {
            DIARIES -> db.update(DiaryContract.Diary.TABLE_NAME, values, selection, selectionArgs)
            DIARY_ID -> {
                val id = ContentUris.parseId(uri)
                db.update(DiaryContract.Diary.TABLE_NAME, values,
                    "${DiaryContract.Diary.COLUMN_ID} = ?", arrayOf(id.toString()))
            }
            else -> throw IllegalArgumentException("Unknown URI: $uri")
        }
        if (rows > 0) context?.contentResolver?.notifyChange(uri, null)
        return rows
    }

    // 5. delete：删除数据
    // 官方提醒：返回删除的行数
    override fun delete(uri: Uri, selection: String?, selectionArgs: Array<out String>?): Int {
        val db = dbHelper.writableDatabase
        val rows = when (uriMatcher.match(uri)) {
            DIARIES -> db.delete(DiaryContract.Diary.TABLE_NAME, selection, selectionArgs)
            DIARY_ID -> {
                val id = ContentUris.parseId(uri)
                db.delete(DiaryContract.Diary.TABLE_NAME,
                    "${DiaryContract.Diary.COLUMN_ID} = ?", arrayOf(id.toString()))
            }
            else -> throw IllegalArgumentException("Unknown URI: $uri")
        }
        if (rows > 0) context?.contentResolver?.notifyChange(uri, null)
        return rows
    }

    // 6. getType：返回 MIME 类型
    // 官方提醒：必须实现！告诉调用者这是什么类型的数据
    override fun getType(uri: Uri): String {
        return when (uriMatcher.match(uri)) {
            DIARIES -> DiaryContract.Diary.CONTENT_TYPE
            DIARY_ID -> DiaryContract.Diary.CONTENT_ITEM_TYPE
            else -> throw IllegalArgumentException("Unknown URI: $uri")
        }
    }

    companion object {
        val uriMatcher = UriMatcher(UriMatcher.NO_MATCH).apply {
            addURI(DiaryContract.AUTHORITY, "diary", DIARIES)
            addURI(DiaryContract.AUTHORITY, "diary/#", DIARY_ID)
        }
    }
}
```

"六个方法，"希尔扳着手指头，"query、insert、update、delete、getType、onCreate。一个都不能少。"

### MIME 类型：告诉系统这是什么

```kotlin
// MIME 类型格式（官方标准）
```

### Contract Class：Provider 的使用说明书

"DiaryContract 是谁？"洛芙问。

```kotlin
// DiaryContract.kt - Provider 的使用说明书（官方推荐）
object DiaryContract {
    // 官方推荐：一个 Provider 通常只有一个 authority
    const val AUTHORITY = "com.campapp.provider"
    val BASE_CONTENT_URI: Uri = Uri.parse("content://$AUTHORITY")

    object Diary {
        const val PATH = "diary"
        val CONTENT_URI: Uri = Uri.withAppendedPath(BASE_CONTENT_URI, PATH)
        const val TABLE_NAME = "diary"

        // 列名（官方推荐：使用 BaseColumns._ID 作为主键列名）
        object Columns : BaseColumns {
            const val COLUMN_ID = "_id"
            const val COLUMN_TITLE = "title"
            const val COLUMN_CONTENT = "content"
            const val COLUMN_MOOD = "mood"
            const val COLUMN_WEATHER = "weather"
            const val COLUMN_LOCATION = "location"
            const val COLUMN_CREATED_AT = "created_at"
            const val COLUMN_UPDATED_AT = "updated_at"
        }

        // MIME 类型
        const val CONTENT_TYPE = "vnd.android.cursor.dir/vnd.$AUTHORITY.$PATH"
        const val CONTENT_ITEM_TYPE = "vnd.android.cursor.item/vnd.$AUTHORITY.$PATH"
    }
}
```

### 第四步：注册到 AndroidManifest

"代码写完了，现在要让系统知道我们这个 Provider。"黛琳的表情认真起来，"不然系统理都不理你。"

```xml
<!-- AndroidManifest.xml -->
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.campapp">

    <!-- 官方推荐：定义自定义权限 -->
    <permission
        android:name="com.campapp.permission.READ_DIARY"
        android:protectionLevel="normal" />
    <permission
        android:name="com.campapp.permission.WRITE_DIARY"
        android:protectionLevel="normal" />

    <application ...>
        <!-- 注册 Content Provider -->
        <provider
            android:name=".DiaryProvider"
            android:authorities="com.campapp.provider"
            android:exported="false"
            android:enabled="true"
            <!-- 权限控制 -->
            android:readPermission="com.campapp.permission.READ_DIARY"
            android:writePermission="com.campapp.permission.WRITE_DIARY"
        />
    </application>
</manifest>
```

"exported = false 是什么意思？"洛芙问。

"就是不对外公开，"希尔解释，"只有你自己的 app 能访问。如果要对外开放，得改成 true，还要加权限控制。"

伊莎补充："Android 10 以后，默认 exported 就是 false。所以如果你不写这个属性，系统会认为你不希望被其他人访问——这是一种安全默认。"

### 权限控制：细粒度管理

"Provider 的权限可以很复杂，"黛琳画出层级图：

```
权限层级（从粗到细）：
├── android:permission        // 统一读写权限
├── android:readPermission    // 单独读权限
├── android:writePermission   // 单独写权限
├── <path-permission>        // 路径级权限（更细）
└── android:grantUriPermissions  // 临时授权
```

"临时授权？"洛芙好奇。

"比如你的邮件 app 要分享一个附件给图片查看器，"黛琳解释，"你不需要给图片查看器永久权限，只需要临时授权那个文件的 URI 就够了。"

```xml
<!-- 开启临时授权 -->
<provider
    android:name=".DiaryProvider"
    android:authorities="com.campapp.provider"
    android:exported="true"
    android:grantUriPermissions="true" />
```

### 第五步：调用 Provider

```kotlin
class MainActivity : AppCompatActivity() {

    private val diaryUri = DiaryContract.Diary.CONTENT_URI

    // 1. 查询所有日记
    fun queryAllDiaries() {
        val projection = arrayOf(
            DiaryContract.Diary.Columns.COLUMN_ID,
            DiaryContract.Diary.Columns.COLUMN_TITLE,
            DiaryContract.Diary.Columns.COLUMN_MOOD
        )

        val cursor = contentResolver.query(
            diaryUri,
            projection,
            null,
            null,
            "${DiaryContract.Diary.Columns.COLUMN_CREATED_AT} DESC"
        )

        cursor?.use {
            while (it.moveToNext()) {
                val id = it.getLong(it.getColumnIndexOrThrow(DiaryContract.Diary.Columns.COLUMN_ID))
                val title = it.getString(it.getColumnIndexOrThrow(DiaryContract.Diary.Columns.COLUMN_TITLE))
                Log.d("Diary", "ID: $id, Title: $title")
            }
        }
    }

    // 2. 插入新日记
    fun addDiary(title: String, content: String, mood: String) {
        val values = ContentValues().apply {
            put(DiaryContract.Diary.Columns.COLUMN_TITLE, title)
            put(DiaryContract.Diary.Columns.COLUMN_CONTENT, content)
            put(DiaryContract.Diary.Columns.COLUMN_MOOD, mood)
        }
        val newUri = contentResolver.insert(diaryUri, values)
        Log.d("Diary", "New diary: $newUri")
    }
}
```

```kotlin
// 批量插入（官方推荐：提升性能）
override fun bulkInsert(uri: Uri, values: Array<out ContentValues>): Int {
    val db = dbHelper.writableDatabase
    var count = 0
    db.beginTransaction()
    try {
        for (v in values) {
            val id = db.insert(DiaryContract.Diary.TABLE_NAME, null, v)
            if (id > 0) count++
        }
        db.setTransactionSuccessful()
    } finally {
        db.endTransaction()
    }
    if (count > 0) context?.contentResolver?.notifyChange(uri, null)
    return count
}
```

### 技术总结

> **创建 Content Provider** —— 本章详细讲解了如何从零创建一个完整的 Content Provider，包括何时需要 Provider、数据存储设计、Content URI 结构、UriMatcher 路由、六个核心方法实现、MIME 类型、Contract Class 编写、AndroidManifest 注册与权限控制。重点理解了 Provider 作为"接口层"的职责分离设计，以及官方推荐的各项最佳实践。

#### 今日关键词

1. **Authority**：Provider 的唯一标识，推荐使用反转域名
2. **UriMatcher**：URI 路由匹配器，# 匹配数字，* 匹配任意字符串
3. **MIME Type**：vnd.android.cursor.dir（目录）vs vnd.android.cursor.item（单项）
4. **Contract Class**：Provider 的使用说明书，定义所有常量
5. **Permissions**：exported、readPermission、writePermission、grantUriPermissions

#### 何时需要 Content Provider

- 向其他应用提供复杂数据或文件
- 允许用户复制数据到其他应用
- 提供自定义搜索建议
- 向桌面组件暴露数据
- 实现 SyncAdapter

#### 官方推荐的六个核心方法

| 方法 | 返回值 | 作用 |
|------|--------|------|
| onCreate() | boolean | 初始化 |
| query() | Cursor | 查询 |
| insert() | Uri | 插入，返回新行 URI |
| update() | int | 更新，返回影响行数 |
| delete() | int | 删除，返回删除行数 |
| getType() | String | 返回 MIME 类型 |

#### 设计思想

- **接口与实现分离**：Provider 只管 CRUD，不管数据怎么存
- **最小权限原则**：默认 exported = false，需要再打开
- **观察者模式**：notifyChange 让数据变化自动传播
- **Contract Class**：集中管理常量，避免拼写错误

3. 实现 query/insert/update/delete/getType。  
4. 注册到 AndroidManifest。

#### Task 2 · URI 设计练习 ★

**目标**：设计完整的 URI 结构。  
**步骤**：
1. 定义 authority 为 `com.mynote.provider`。  
2. 设计 notes（所有笔记）和 notes/#（单条笔记）的 URI。  
3. 用 UriMatcher 实现匹配。  
**验收**：
- [ ] 能匹配列表和单条两种 URI

---

#### Task 3 · MIME 类型实现 ★

**目标**：正确返回 MIME 类型。  
**步骤**：
1. 实现 getType()。  
2. 列表 URI 返回 vnd.android.cursor.dir。  
3. 单条 URI 返回 vnd.android.cursor.item。  
**验收**：
- [ ] 用 adb shell 能看到正确的 MIME

---

#### Task 4 · Contract Class 编写 ★★

**目标**：编写规范的 Contract Class。  
**步骤**：
1. 定义 AUTHORITY 常量。  
2. 定义 CONTENT_URI。  
3. 定义所有列名常量（继承 BaseColumns）。  
4. 定义 MIME 类型。  
**验收**：

#### Task 5 · 权限控制 ★★★

**目标**：添加完整的权限控制。  
**步骤**：
1. 定义 read/write permission。  
2. Provider 声明权限。  
3. 调用方申请权限。  
**验收**：
- [ ] 无权限无法访问

---

#### Task 6 · 批量插入 ★★★

**目标**：实现 bulkInsert。  
**步骤**：
1. 开启事务。  
2. 循环插入。  
3. 成功后 notifyChange。  
**验收**：
- [ ] 100 条插入 < 1 秒

---

#### Task 7 · 临时授权 ★★★★

**目标**：实现临时 URI 授权。  
**步骤**：
1. 设置 android:grantUriPermissions="true"。  
2. 用 IntentSender 授予临时权限。  
3. 对方用 granted URI 访问。  
**验收**：
- [ ] 临时授权有效，过期后失效

---

#### Task 8 · 文件 Provider ★★★★★

**目标**：用 FileProvider 分享文件。  
**步骤**：
1. 继承 FileProvider。  
2. 配置 file_paths XML。  
3. 生成 content:// URI。  
4. 授予临时权限。  
**验收**：
- [ ] 能分享文件给其他 app
