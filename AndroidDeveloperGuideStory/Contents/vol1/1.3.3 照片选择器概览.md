## 1.3.3.1 什么是照片选择器

“照片选择器（Photo Picker）是Android 13引入的新API，”黛琳解释道，“它让用户可以选择特定的照片或视频给App，而不需要授予整个相册的访问权限。”

“对！”黛琳赞同道，“照片选择器就是这样一个'只借几本'的机制。”

## 1.3.3.2 使用照片选择器

“用法非常简单，”黛琳说道。

```kotlin
// Android 13+ 使用照片选择器
fun pickPhotos() {
    val intent = Intent(MediaStore.ACTION_GET_CONTENT).apply {
        type = "image/*"  // 或者 "video/*"
        putExtra(Intent.EXTRA_ALLOW_MULTIPLE, true)  // 允许多选
    }
    startActivityForResult(intent, REQUEST_CODE)
}
```

## 1.3.3.3 兼容旧版本

“如果要兼容旧版本怎么办？”希尔问。

“可以检测系统是否支持照片选择器，”黛琳说道。

```kotlin
// 检测是否支持照片选择器
fun isPhotoPickerAvailable(): Boolean {
    return if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
        true  // Android 13+ 原生支持
    } else {
        // Android 12及以下，回退到旧方式
        // 可以用ActivityResultContracts.GetContent()
        false
    }
}

// 使用兼容方式
```

## 1.3.3.4 注意事项

“使用照片选择器有几个注意点，”黛琳强调道。

1. **不需要存储权限** - 照片选择器不需要任何存储权限
2. **Uri是临时的** - 如果需要长期访问，需要获取持久权限
3. **只读访问** - App只能读取，不能修改或删除

```kotlin
// 获取持久访问权限
fun takePersistablePermission(uri: Uri) {
    val takeFlags = Intent.FLAG_GRANT_READ_URI_PERMISSION
    try {
        contentResolver.takePersistableUriPermission(uri, takeFlags)
    } catch (e: SecurityException) {
        // 权限获取失败
    }
}
```

## 1.3.3.5 专业技术总结

本章我们学习了照片选择器。

**核心要点：**

1. **照片选择器是Android 13+的新API** - 用户选择特定照片给App
2. **不需要存储权限** - 保护用户隐私
3. **使用简单** - 用Intent调用
4. **支持单选和多选**
5. **注意Uri的持久化**

**选择器 vs 权限：**

| 方式 | 优点 | 缺点 |
|------|------|------|
| 照片选择器 | 简单、不需要权限 | 每次都要选 |
| 存储权限 | 一次授权，一直可用 | 权限过大 |
