# 1.7.2 新一代数据存储DataStore

“除了SharedPreferences和Room，还有一个新选择——DataStore，”黛琳说道。

洛FR问：“DataStore是什么？和SharedPreferences有什么区别？”

## 1.7.2.1 DataStore是什么

“DataStore是Jetpack推出的新一代数据存储方案，”黛琳解释道，“它是SharedPreferences的替代品，但更强大。”

**DataStore的特点：**
- 异步API（基于Flow）
- 类型安全
- 支持Proto DataStore

## 1.7.2.2 Preferences DataStore

“DataStore有两种类型，第一种是Preferences DataStore，”黛琳说道。

```kotlin
// 添加依赖
dependencies {
    val datastoreVersion = "1.0.0"
    implementation("androidx.datastore:datastore-preferences:$datastoreVersion")
}
```

```kotlin
// 使用
class DataStoreManager(private val context: Context) {
    
    private val Context.dataStore by preferencesDataStore(name = "settings")
    
    // 保存
    suspend fun saveSettings(settings: Settings) {
        context.dataStore.edit { prefs ->
            prefs[stringPreferencesKey("username")] = settings.username
            prefs[booleanPreferencesKey("dark_mode")] = settings.darkMode
        }
    }
    
    // 读取 - 返回Flow
    val settingsFlow: Flow<Settings> = context.dataStore.data.map { prefs ->
        Settings(
            username = prefs[stringPreferencesKey("username")] ?: "",
            darkMode = prefs[booleanPreferencesKey("dark_mode")] ?: false
        )
    }
}
```

## 1.7.2.3 Proto DataStore

“第二种是Proto DataStore，需要定义协议缓冲区，”黛琳继续说道。

```protobuf
// user.proto
syntax = "proto3";

message User {
  string name = 1;
  int32 age = 2;
}
```

```kotlin
// 生成Serializer
class UserSerializer : Serializer<User> {
    override val defaultValue: User = User.getDefaultInstance()
    
    override suspend fun readFrom(input: InputStream): User {
        try {
            return User.parseFrom(input)
        } catch (e: Exception) {
            return defaultValue
        }
    }
    
    override suspend fun writeTo(t: User, output: OutputStream) {
        t.writeTo(output)
    }
}
```

```kotlin
// 使用Proto DataStore
private val Context.userDataStore: DataStore<User> by dataStore(
    fileName = "user.pb",
    serializer = UserSerializer()
)

// 保存
suspend fun saveUser(user: User) {
    context.userDataStore.updateData { user }
}

// 读取
val userFlow: Flow<User> = context.userDataStore.data
```

## 1.7.2.4 DataStore vs SharedPreferences

“有什么区别呢？”洛FR问。

| 特性 | SharedPreferences | DataStore |
|------|-------------------|-----------|
| API | 同步 | 异步(Flow) |
| 类型安全 | 否 | Proto版本是 |
| 迁移支持 | 简单 | 有迁移API |
| 性能 | 可能会阻塞 | 不会阻塞 |

## 1.7.2.5 专业技术总结

本章我们学习了DataStore。

**核心要点：**

1. **DataStore是SharedPreferences的替代** - Jetpack推荐
2. **两种类型** - Preferences和Proto
3. **异步API** - 基于Flow，不会阻塞
4. **类型安全** - Proto DataStore提供
5. **迁移支持** - 可以从SharedPreferences迁移

**选择：**

| 场景 | 推荐 |
|------|------|
| 简单key-value | SharedPreferences |
| 复杂数据 | DataStore Preferences |
| 需要类型安全 | Proto DataStore |
| 大量数据 | Room |
