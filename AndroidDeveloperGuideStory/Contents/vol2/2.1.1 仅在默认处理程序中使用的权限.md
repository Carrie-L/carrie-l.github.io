# 2.1.1 仅在默认处理程序中使用的权限

清晨的光线从帐篷缝隙里钻进来，带着一种初夏特有的温热。

洛芙翻了个身，脸颊蹭到睡袋边缘粗糙的缝线。她迷迷糊糊地听见帐篷外面有鸟叫——不是春天那种细碎的啾啾声，而是"喳——喳——"的大嗓门儿，像在宣告什么季节开始了。

"唔……几点了……"

她把手机从枕头底下摸出来，屏幕亮起来的瞬间眯起眼睛。电量显示百分之六十三，信号只有一格。洛芙坐起身，掀开帐篷帘子往外探脑袋——

希尔正盘腿坐在草地上，手机贴在耳边，眉头皱成一团。

"……对，我这边信号不太好，你能再说一遍吗？喂？喂——"

她烦躁地把手机从耳边拿下来，盯着屏幕看了一眼，然后朝帐篷方向大喊："洛芙！你的手机信号怎么样？"

洛芙低头看了看自己手机上那个孤零零的信号格，摇了摇头。

"也是一格……怎么回事啊，昨天还好好的。"

希尔站起身，拍了拍裤子上的草屑，快步走到洛芙帐篷边上蹲下来。晨光把她乱蓬蓬的短发照得有点发亮——她昨晚肯定又没好好梳头。洛芙忍不住伸手在希尔的脑袋顶上揉了一把，希尔没躲，只是用胳膊肘轻轻顶了她一下。

"别闹，我在想事情呢。"希尔说，"你说，这手机信号的事……会不会跟咱们搭营地的位置有关系？"

"跟搭营地位置有关系？"洛芙歪着脑袋。

"就是……"希尔正要说下去，远处传来一阵窸窸窣窣的声响。伊莎从她和黛琳合住的帐篷里钻出来，头发上还别着一片不知道什么时候粘上去的小叶子，整个人散发着一股刚睡醒的慵懒气息。

"早啊。"伊莎打了个哈欠，朝两人走过来，"你们俩怎么这么早就开始开会了？"

"希尔在分析手机信号的问题。"洛芙说。

"哦？"伊莎眨了眨眼睛，"信号不好啊……让我想想……"

她歪着头看了看远处的山峦，又看了看周围树林的分布，若有所思地说："你们说，会不会是因为这里离最近的基站太远了？"

"不是这个问题。"希尔摇头，"我刚才打电话给客服问了，我们现在的位置虽然偏僻，但基站信号应该是覆盖到的。问题是……"

她顿了顿，把手机屏幕转过来给两人看。

"看这个——我的手机显示我是'未注册的本机拨号应用'，很多功能都用不了。"

洛芙凑过去看，屏幕上有一行小字写着"未注册的本机拨号应用"，旁边还画了一个小小的警告图标。

"这是什么意思啊？"洛芙问。

希尔正要回答，黛琳的声音从帐篷里传来："意思就是你的手机没有把希尔这个拨号应用注册成系统默认的拨号处理程序。"

黛琳掀开帘子走出来，手里还捧着一个保温杯，热气从杯口袅袅升起。她的表情一如既往地平静，像是早就料到了这场对话。

"黛琳姐！"希尔眼睛一亮，"你怎么知道的？"

"昨晚我想发一条短信，发现也遇到了类似的问题。"黛琳抿了一口热可可，"系统提示我'此应用不是默认短信应用'，很多操作都受限了。"

她把手机递过来，屏幕上显示着一行灰色的提示文字：*"此应用不是默认短信应用，无法执行此操作。"*

"你看，这里写的是'无法执行此操作'，但不是'没有这个权限'。"黛琳用保温杯指了指那行字，"区别很大。"

洛芙看着那行字，脑子里好像有点明白又不完全明白。

"等等，"她举起手，"我有点跟不上——你说'不是没有这个权限'，但它明明显示'无法执行'啊？"

"这个问题问得好。"黛琳点点头，"走，去找块平整的石头坐下说，这事情解释起来有点复杂。"

四个人在营地边缘一块被晨光晒得微微发暖的大石头上坐下。希尔把笔记本电脑架在膝盖上，屏幕朝向大家。黛琳喝了一口热可可，开始慢慢地说。

"在我们讲手机信号和拨号的事情之前，我想先问你们一个问题——"黛琳看向洛芙和希尔，"你们觉得，一个短信应用，和一个普通的游戏应用，在系统眼里，有什么本质区别？"

洛芙想了想，摇摇头。

"我觉得……游戏应用是自己玩自己的，不用管别人？"

"差不多对了。"黛琳微微一笑，"更准确地说——有些应用需要成为系统的'默认处理程序'，才能正常工作。"

她在希尔笔记本电脑旁边的白纸上画了一个简单的示意图。

"我们先来说一个你们都熟悉的东西——浏览器。"

黛琳画了一个带着"浏览器"标签的方框，然后在旁边画了另一个方框写着"用户点击的链接"。

"当用户在某个应用里点击一个链接的时候，系统会把用户点击的这个链接'交给'一个浏览器来处理。"她用笔尖点了点那两个方框，"但是，系统里有好几个浏览器——Chrome、Firefox、Opera，还有手机自带的默认浏览器。系统怎么知道该交给哪个浏览器呢？"

"看用户把哪个设成默认浏览器？"希尔说。

"对。"黛琳在两个浏览器方框之间画了一条虚线，虚线连着"默认"两个字，"系统只把链接交给被用户设置为默认的那个浏览器。这个浏览器就是'默认处理程序'。"

"那短信、电话，是不是也是一样的道理？"洛芙问。

"完全正确。"黛琳的语气里带着一点赞许，"短信、电话、拨号，这些功能都是'核心设备功能'——系统认为这些功能太重要了，不能随便哪个应用都能插手。"

她把白纸翻到新的一面，开始画第二个图。

"看这张图。"

```
┌──────────────────────────────────────────────────────┐
│                    Android 系统                       │
│                                                       │
│   ┌─────────────┐        ┌─────────────┐             │
│   │  短信应用 A  │        │  短信应用 B  │             │
│   │ (默认处理程序) │        │             │             │
│   └──────┬──────┘        └─────────────┘             │
│          │                                             │
│          │  当用户收到短信时                            │
│          ▼                                             │
│   系统只把短信交给 A，因为 A 是默认处理程序              │
│                                                       │
└──────────────────────────────────────────────────────┘
```

"在这张图里，有两个应用都想处理短信。应用 A 是默认处理程序，应用 B 不是。当系统收到一条新短信的时候，它只会把短信交给应用 A——因为只有应用 A 才被用户信任，可以读取、发送、管理用户的短信。"

"原来是这样……"洛芙慢慢点头，"那应用 B 就完全收不到短信了吗？"

"不是收不到——是系统根本不把短信'投递'给它。"黛琳说，"这就像你订了一份报纸，报社只会把报纸送到你指定的地址，不会同时送到你邻居家。虽然邻居家也有一个邮箱，但那个邮箱不被报社信任。"

希尔听到这个比喻，忍不住噗嗤一声笑了出来。

"这个比喻好奇怪——邻居的邮箱不信任报社？"

"现实中确实很奇怪，但在 Android 系统里，这是保护用户隐私的核心机制。"黛琳的嘴角也微微扬起，"你想啊——短信里可能有机密的银行验证码、私密对话、甚至隐私信息。如果随便哪个应用都能读取短信，那岂不是很不安全？"

洛芙突然想到了什么，倒吸一口凉气。

"等等——那岂不是很多应用都想成为'默认处理程序'？这样就能随便读取用户的短信了？"

"没错。"黛琳点头，"所以 Android 系统给'默认处理程序'设了一个很高的门槛。不是你想当就能当的——你必须证明你有能力承担这个责任。"

"怎么证明？"希尔问。

"首先，你必须在代码里声明你要承担这个责任——通过在 AndroidManifest.xml 里添加特定的 intent-filter。"黛琳说，"其次，你还必须在 manifest 里申请相应的权限。"

她喝了一口热可可，继续说道：

"但这里有一个很关键的限制——某些权限，只有当你声明了'我要成为某某功能的默认处理程序'的时候，系统才允许你申请。"

"比如？"洛芙追问。

"比如 READ_SMS、WRITE_SMS、READ_CALL_LOG、WRITE_CALL_LOG，这些权限。"黛琳说，"如果你没有声明你要成为默认短信应用或默认拨号应用，这些权限你是根本申请不到的。"

希尔立刻打开电脑，在 Android 开发者文档里搜了一下，然后把屏幕转过来给大家看。

"我找到了——官方文档把这个叫 'Permissions used only in default handlers'，翻译过来就是'仅在默认处理程序中使用的权限'。"

屏幕上显示着一段话：

> "Your app must be able to perform the functionality for which it's a default handler. For example, a default SMS handler must be able to send and receive text messages."

"看这句话。"黛琳指着屏幕，"文档说，如果你要成为某个功能的默认处理程序，你必须真正具备那个功能的能力。举个例子——如果你要成为默认短信应用，你必须能够发送和接收短信。"

"这不是废话吗？"希尔挠挠头，"不想发送短信的话，当什么短信应用啊？"

"这不是废话，这是一个很重要的原则。"黛琳摇摇头，"它的意思是——你不能一边当默认短信应用，一边又偷偷摸摸地只读取短信、不发送短信。系统要求默认处理程序必须是'完整'的实现。"

"完整实现……"洛芙把这四个字默念了一遍。

"对。"黛琳点头，"系统相信默认处理程序有能力处理某类任务，所以把整个任务都交给它。如果这个应用只做一半，系统就不知道该怎么分配任务了。"

她重新拿起白纸，开始画第三张图。

"我们再来说一个具体的例子——通话记录。"

```
┌──────────────────────────────────────────────────────┐
│                    通话记录权限                        │
│                                                       │
│   READ_CALL_LOG / WRITE_CALL_LOG                      │
│                                                       │
│   ⚠️ 仅限以下应用使用：                                 │
│   1. 默认拨号应用（系统默认的电话）                     │
│   2. 默认短信应用（系统默认的短信）                     │
│   3. 运营商应用（Carrier apps）                        │
│   4. 设备管理应用（Device admin apps）                 │
│                                                       │
│   ❌ 普通应用即使在 manifest 里声明了，                 │
│      系统也会在安装时拒绝授予这些权限！                 │
│                                                       │
└──────────────────────────────────────────────────────┘
```

"这个图显示了 READ_CALL_LOG 和 WRITE_CALL_LOG 这两个权限的特殊性。"黛琳说，"普通应用如果在 manifest 里声明这两个权限，系统会在安装时直接拒绝授予——根本不给你请求的机会。"

"那岂不是安装了也用不了？"洛芙问。

"对。"黛琳说，"系统会把这两个权限的授予状态设为'拒绝'，无论用户怎么操作都无法改变。因为从系统安全的角度来看，普通应用没有理由去读取用户的通话记录——那是高度敏感的个人隐私数据。"

"但是……"希尔举手，"有些应用确实需要读取通话记录啊？比如那种手机找回应用，或者家长控制应用？"

"好问题。"黛琳点头，"所以系统给了一个例外——如果你的应用确实需要使用这些权限，你必须先向系统证明你有'正当理由'。在 Android 系统中，这个证明的过程就是'声明你要成为默认处理程序'。"

"具体怎么操作？"希尔已经开始在键盘上敲敲打打了。

"在 AndroidManifest.xml 里，用 `<intent-filter>` 和 `<queries>` 标签来声明你的应用要处理哪些类型的 Intent。"黛琳说，"以短信应用为例——"

她拿过希尔的电脑，在键盘上噼里啪啦地输入了一段代码示例：

```xml
<!-- AndroidManifest.xml -->
<manifest>
    <!-- 其他权限... -->

    <!-- 如果要成为默认短信应用，必须声明这个权限 -->
    <uses-permission android:name="android.permission.READ_SMS" />
    <uses-permission android:name="android.permission.RECEIVE_SMS" />
    <uses-permission android:name="android.permission.SEND_SMS" />

    <!-- 声明应用要处理的 Intent -->
    <queries>
        <!-- 声明你的应用能响应短信相关的 Intent -->
        <intent>
            <action android:name="android.intent.action.MAIN" />
            <category android:name="android.intent.category.MAIN" />
        </intent>
        <intent>
            <action android:name="android.intent.action.RECEIVE_SMS" />
        </intent>
        <intent>
            <action android:name="android.intent.action.SEND_SMS" />
        </intent>
    </queries>

    <!-- 同时，你的应用必须声明自己是某种 Role 的候选人 -->
    <application>
        <activity
            android:name=".MainActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>
</manifest>
```

"等等，"洛芙凑近屏幕看了看，"这里有个`<queries>`标签是什么意思？"

"`<queries>`是 Android 11（API 30）引入的新机制。"黛琳解释道，"以前，应用可以查询手机上安装了哪些其他应用——但这样会泄露用户隐私，比如可以知道用户有没有安装某些'敏感'应用。所以 Google 把这个查询能力限制了。"

她顿了顿，喝了一口热可可。

"现在，如果你想查询某个 Intent 能不能被处理，你必须在`<queries>`里声明你要查询哪些 Intent。系统会检查——如果你没有声明，系统就不会告诉你。"

"好严格……"洛芙小声嘀咕。

"这是为了保护用户隐私。"黛琳说，"Android 越来越重视隐私保护了，很多以前'随便用'的能力，现在都需要显式声明。"

希尔举手打断了对话："那具体怎么申请成为'默认处理程序'呢？光在 manifest 里声明够不够？"

"光声明是不够的。"黛琳摇头，"用户还必须主动把你的应用设为默认。这是一个主动授权的过程——只有用户自己才能决定信任哪个应用来处理他的短信和电话。"

"那应用怎么知道用户有没有把自己设为默认？"希尔问。

"问得好。"黛琳微微一笑，"Android 提供了一个叫'Role Manager'的 API，可以让应用查询自己有没有被授予某个 Role。"

她拿过希尔的电脑，开始写代码：

```kotlin
// 检查应用是否是默认短信应用
// 需要先在 AndroidManifest.xml 里声明 READ_SMS 权限
// 但在 API 30+，普通应用即使声明了也不会被授予
// 必须先通过 RoleManager 申请 ROLE_SMS

import android.app.role.RoleManager
import android.content.Intent
import android.os.Build

class DefaultAppChecker {
    
    fun isDefaultSmsApp(context: Context): Boolean {
        val roleManager = context.getSystemService(RoleManager::class.java)
        
        // 检查应用是否已经拥有 ROLE_SMS
        // 注意：这个检查在 API 30+ 才有意义
        return if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R) {
            roleManager.isRoleAvailable(RoleManager.ROLE_SMS) &&
            roleManager.isRoleHeld(RoleManager.ROLE_SMS)
        } else {
            // API 30 之前的旧检查方式
            val packageManager = context.packageManager
            val componentName = ComponentName(context, DefaultSmsActivity::class.java)
            packageManager.resolveActivity(
                Intent(Intent.ACTION_MAIN).apply {
                    addCategory(Intent.CATEGORY_LAUNCHER)
                },
                PackageManager.MATCH_DEFAULT_ONLY
            )?.activityInfo?.packageName == context.packageName
        }
    }
    
    fun requestDefaultSmsRole(activity: Activity) {
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R) {
            val roleManager = activity.getSystemService(RoleManager::class.java)
            
            // 检查角色是否可用（某些设备可能不支持）
            if (roleManager.isRoleAvailable(RoleManager.ROLE_SMS)) {
                // 检查应用是否已经拥有这个角色
                if (!roleManager.isRoleHeld(RoleManager.ROLE_SMS)) {
                    // 请求用户授予角色
                    // 这会打开系统设置页面，让用户手动确认
                    val intent = roleManager.createRequestRoleIntent(RoleManager.ROLE_SMS)
                    activity.startActivityForResult(intent, REQUEST_CODE_MANAGE_DEFAULT_SMS)
                }
            }
        }
    }
    
    companion object {
        private const val REQUEST_CODE_MANAGE_DEFAULT_SMS = 1
    }
}
```

"哇……代码好多。"洛芙看着屏幕，感觉有点头晕。

"我来给你们解释一下这段代码在做什么。"希尔敲了敲键盘，把洛芙的注意力拉回来，"首先——"

她指着屏幕上的第一段函数：

"`isDefaultSmsApp()` 这个函数检查当前应用是不是默认短信应用。怎么检查呢？它调用了系统服务 `RoleManager`，问系统——'我有没有持有 ROLE_SMS 这个角色？'"

"如果系统说'有'，那应用就是默认短信应用；如果系统说'没有'，那就不是。"

"那 `requestDefaultSmsRole()` 呢？"洛芙问。

"这个函数用于向系统申请成为默认短信应用。"希尔说，"它会调用 `createRequestRoleIntent()` 创建一个特殊的 Intent，这个 Intent 会打开系统设置页面，让用户自己决定要不要把你的应用设为默认。"

"等等——"洛芙突然想到了什么，"也就是说，应用不能自己把自己变成默认，必须用户同意？"

"对。"希尔点头，"这是 Android 系统的安全机制。任何想把用户的重要数据（短信、通话记录等）交给第三方应用的操作，都必须经过用户明确同意。"

伊莎一直在旁边安静地听着，这时候突然开口了：

// MainActivity.kt
package com.campingteam.smshandler

import android.Manifest
import android.app.role.RoleManager
import android.content.Context
import android.content.Intent
import android.content.pm.PackageManager
import android.os.Build
import android.os.Bundle
import android.widget.Button
import android.widget.TextView
import androidx.activity.result.contract.ActivityResultContracts
import androidx.appcompat.app.AppCompatActivity
import androidx.core.content.ContextCompat

class MainActivity : AppCompatActivity() {
    
    private lateinit var statusText: TextView
    private lateinit var setDefaultButton: Button
    
    // 用于接收权限请求结果的 launcher
    private val requestPermissionLauncher = registerForActivityResult(
        ActivityResultContracts.RequestMultiplePermissions()
    ) { permissions ->
        // 检查权限是否全部授予
        val allGranted = permissions.all { it.value }
        if (allGranted) {
            statusText.text = "✓ 短信权限已授予"
        } else {
            statusText.text = "✗ 部分权限被拒绝"
        }
    }
    
    // 用于接收角色申请结果的 launcher
    private val requestRoleLauncher = registerForActivityResult(
        ActivityResultContracts.StartActivityForResult()
    ) { result ->
        if (isDefaultSmsApp()) {
            statusText.text = "✓ 已成为默认短信应用"
            setDefaultButton.isEnabled = false
        } else {
            statusText.text = "✗ 未成为默认短信应用"
        }
    }
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        
        statusText = findViewById(R.id.status_text)
        setDefaultButton = findViewById(R.id.set_default_button)
        
        // 检查当前状态
        updateStatus()
        
        // 设置默认应用按钮
        setDefaultButton.setOnClickListener {
            requestDefaultSmsRole()
        }
    }
    
    override fun onResume() {
        super.onResume()
        // 每次回到前台都检查一次状态
        updateStatus()
    }
    
    private fun updateStatus() {
        // 检查权限
        val hasSmsPermission = ContextCompat.checkSelfPermission(
            this, Manifest.permission.READ_SMS
        ) == PackageManager.PERMISSION_GRANTED
        
        // 检查是否默认短信应用

        val statusMessage = buildString {
            append("短信权限: ")
            append(if (hasSmsPermission) "✓ 已授予" else "✗ 未授予")
            append("\n")
            append("默认短信应用: ")
            append(if (isDefault) "✓ 是" else "✗ 否")
        }
        
        statusText.text = statusMessage
        
        // 根据状态决定按钮文字
        setDefaultButton.isEnabled = !isDefault
        setDefaultButton.text = if (isDefault) "已设为默认" else "设为默认短信应用"
    }
    
    /**
     * 检查应用是否是默认短信应用
     * 
     * API 30+ 使用 RoleManager 检查
     * API 30 之前使用 PackageManager 检查默认 Activity
     */
    private fun isDefaultSmsApp(): Boolean {
        return if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R) {
            val roleManager = getSystemService(Context.ROLE_SERVICE) as RoleManager
            roleManager.isRoleAvailable(RoleManager.ROLE_SMS) &&
            roleManager.isRoleHeld(RoleManager.ROLE_SMS)
        } else {
            // API 30 之前的旧方法：通过检查默认的 SMS Intent Handler
            val packageManager = packageManager
            val intent = Intent(Intent.ACTION_MAIN).apply {
                addCategory(Intent.CATEGORY_LAUNCHER)
            }
            val resolveInfo = packageManager.resolveActivity(
                intent,
                PackageManager.MATCH_DEFAULT_ONLY
            )
            resolveInfo?.activityInfo?.packageName == packageName
        }
    }
    
    /**
     * 请求成为默认短信应用
     * 
     * 这会启动系统设置页面，让用户手动确认
     * 用户可以选择'总是'或'仅本次'
     */
    private fun requestDefaultSmsRole() {
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R) {
            val roleManager = getSystemService(Context.ROLE_SERVICE) as RoleManager
            
            if (roleManager.isRoleAvailable(RoleManager.ROLE_SMS)) {
                if (!roleManager.isRoleHeld(RoleManager.ROLE_SMS)) {
                    val intent = roleManager.createRequestRoleIntent(RoleManager.ROLE_SMS)
                    requestRoleLauncher.launch(intent)
                } else {
                    // 已经有角色了，检查权限
                    checkAndRequestSmsPermissions()
                }
            } else {
                statusText.text = "✗ 当前设备不支持默认短信应用"
            }
        } else {
            // API 30 之前的设备，直接请求权限
            checkAndRequestSmsPermissions()
        }
    }
    
    /**
     * 检查并请求短信权限
     * 
     * 注意：对于 API 33+，READ_SMS 等权限属于'受限权限'
     * 即使在 manifest 里声明了，也需要通过特殊渠道申请
     * 普通应用（非默认短信应用）无法获取这些权限
     */
    private fun checkAndRequestSmsPermissions() {
        val permissionsToRequest = mutableListOf<String>()
        
        // 检查各个权限的状态
        if (ContextCompat.checkSelfPermission(this, Manifest.permission.READ_SMS)
            != PackageManager.PERMISSION_GRANTED
        ) {
            permissionsToRequest.add(Manifest.permission.READ_SMS)
        }
        
        if (ContextCompat.checkSelfPermission(this, Manifest.permission.RECEIVE_SMS)
            != PackageManager.PERMISSION_GRANTED
        ) {
            permissionsToRequest.add(Manifest.permission.RECEIVE_SMS)
        }
        
        if (ContextCompat.checkSelfPermission(this, Manifest.permission.SEND_SMS)
            != PackageManager.PERMISSION_GRANTED
        ) {
            permissionsToRequest.add(Manifest.permission.SEND_SMS)
        }
        
        if (permissionsToRequest.isNotEmpty()) {
            requestPermissionLauncher.launch(permissionsToRequest.toTypedArray())
        } else {
            statusText.text = "✓ 所有短信权限已授予"
        }
    }
}

package com.campingteam.smshandler

import android.content.BroadcastReceiver
import android.content.Context
import android.content.Intent
import android.provider.Telephony
import android.util.Log

/**
 * 短信接收器
 * 
 * 用于接收系统广播的 SMS 消息
 * 注意：这个 receiver 必须在 manifest 里注册
 * 并且 exported 必须设为 true
 * 
 * @see AndroidManifest.xml 中的 receiver 配置
 */
class SmsReceiver : BroadcastReceiver() {
    
    companion object {
        private const val TAG = "SmsReceiver"
    }
    
    override fun onReceive(context: Context, intent: Intent) {
        // 检查 Intent 的 Action
        if (intent.action != Telephony.Sms.Intents.SMS_RECEIVED_ACTION) {
            Log.w(TAG, "Received unknown intent: ${intent.action}")
            return
        }
        
        // 读取短信内容
        val messages = Telephony.Sms.Intents.getMessagesFromIntent(intent)
        
        if (messages.isNullOrEmpty()) {
            Log.w(TAG, "No messages in intent")
            return
        }
        
        // 处理每一条短信
        for (message in messages) {
            val sender = message.originatingAddress  // 发送者号码
            val body = message.messageBody            // 短信内容
            val timestamp = message.timestampMillis   // 时间戳
            
            Log.d(TAG, "收到短信 - 来自: $sender, 内容: $body")
            
            // 在这里可以添加你的业务逻辑
            // 比如：
            // - 保存到数据库
            // - 显示通知
            // - 自动提取验证码
            // 等等
        }
    }
}

## 今日关键词

**默认处理程序（Default Handler）**：在 Android 系统中，被用户指定处理某类特定任务的应用程序。比如默认短信应用负责处理所有短信相关的操作。

**角色（Role）**：Android 11 引入的安全机制，用于管理敏感权限的授予。只有持有相应角色的应用才能申请特定的敏感权限。

**RoleManager**：系统服务，用于查询应用是否持有某个角色，以及申请成为某个角色的默认处理程序。

**受限权限（Restricted Permission）**：Android 13 引入的特殊权限类型，需要在 Google Play Console 中额外申请才能获得。

**Intent Filter**：在 manifest 中声明的过滤器，用于告诉系统你的应用能够处理哪些类型的 Intent。

**`<queries>`**：Android 11 引入的 manifest 配置，用于声明你的应用需要查询哪些 Intent 的处理能力。

**BROADCAST_SMS**：系统权限，只有持有此权限的应用才能接收短信相关的系统广播。

**SMS Token API**：Android 提供的验证码自动填充 API，允许应用在用户授权的情况下自动读取短信验证码，而无需读取整条短信内容。

**NotificationListenerService**：系统服务，允许应用读取通知栏的通知内容，是短信权限的替代方案之一。

**单一职责原则（Single Responsibility Principle）**：软件设计原则，认为每个组件应该只负责一件事。在 BroadcastReceiver 中体现为：Receiver 只负责接收广播，把业务逻辑交给其他组件处理。
