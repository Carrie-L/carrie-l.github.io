# 7.1.10 山巅的类型之舞

清晨，露营编程旅团的姑娘们爬到了山顶。站在高处，整个山谷笼罩在白色的晨雾中，宛如仙境。

“好美啊！”伊莎感叹道。

“这让我想起了前台服务的各种类型，”黛琳笑道，“每一种都像这雾气一样，各有各的特点。”

## 7.1.10.1 前台服务类型概述

“在Android 12之前，前台服务只有一个通用类型，”黛琳说道，“但从Android 12开始，Google把前台服务分成了很多种类型。”

她扳着手指数道：

- **mediaPlayback** - 媒体播放
- **location** - 位置追踪
- **camera** - 摄像头使用
- **microphone** - 麦克风使用
- **phoneCall** - 通话中
- **dataSync** - 数据同步
- **mediaProjection** - 屏幕投影
- **remoteMessaging** - 远程消息

“每种类型都需要在Manifest中声明，”黛琳补充道，“而且需要的权限也不一样。”

## 7.1.10.2 mediaPlayback - 媒体播放

“第一个是`mediaPlayback`，最常用的类型，”黛琳说道。

```xml
<!-- Manifest声明 -->
<service
    android:name=".MusicService"
    android:foregroundServiceType="mediaPlayback"
    android:exported="false" />
```

```kotlin
// 需要的权限
// Android 9+
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />

<!-- Android 13+ -->
<uses-permission android:name="android.permission.FOREGROUND_SERVICE_MEDIA_PLAYBACK" />
```

“什么情况下用这个？”洛FR问。

“播放音乐、视频，或者任何需要持续播放媒体内容的时候，”黛琳解释道，“这是最常用的前台服务类型。”

## 7.1.10.3 location - 位置追踪

“第二个是`location`，”黛琳说道，“做导航或健身追踪应用时用。”

```xml
<service
    android:name=".LocationService"
    android:foregroundServiceType="location"
    android:exported="false" />
```

```kotlin
// 需要的权限
// Android 9+
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />

<!-- Android 10+ 后台位置 -->
<uses-permission android:name="android.permission.ACCESS_BACKGROUND_LOCATION" />

<!-- Android 12+ -->
<uses-permission android:name="android.permission.FOREGROUND_SERVICE_LOCATION" />
```

“位置服务比较特殊，”黛琳强调道，“即使声明了这个类型，如果用户没有授予位置权限，服务也启动不了。”

## 7.1.10.4 camera - 摄像头

“第三个是`camera`，”黛琳说道，“需要使用摄像头时用。”

```xml
<service
    android:name=".CameraService"
    android:foregroundServiceType="camera"
    android:exported="false" />
```

```kotlin
// 需要的权限
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
<uses-permission android:name="android.permission.CAMERA" />

<!-- Android 13+ 还需要动态检查 -->
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
    if (checkSelfPermission(Manifest.permission.CAMERA) != 
        PackageManager.PERMISSION_GRANTED) {
        // 不能启动
    }
}
```

“什么场景会用到？”伊莎问。

“比如拍摄长时间的视频，或者视频通话，”黛琳解释道，“这些场景需要摄像头一直工作，用户需要知道摄像头正在被使用。”

## 7.1.10.5 microphone - 麦克风

“第四个是`microphone`，”黛琳说道，“录音时用。”

```xml
<service
    android:name=".RecordingService"
    android:foregroundServiceType="microphone"
    android:exported="false" />
```

```kotlin
// 需要的权限
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
<uses-permission android:name="android.permission.RECORD_AUDIO" />

<!-- Android 13+ 还需要动态检查 -->
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
    checkSelfPermission(Manifest.permission.RECORD_AUDIO)
}
```

“特别注意，”黛琳严肃地说，“麦克风类型在Android 13上检查特别严格。用户必须明确授权才能启动。”

## 7.1.10.6 其他类型

“剩下的类型用的比较少，”黛琳简单介绍道。

**phoneCall - 通话中：**
```xml
<service
    android:name=".CallService"
    android:foregroundServiceType="phoneCall"
    android:exported="false" />
```
需要`ANSWER_PHONE_CALLS`权限。

**dataSync - 数据同步：**
```xml
<service
    android:name=".SyncService"
    android:foregroundServiceType="dataSync"
    android:exported="false" />
```
用于后台同步数据。

**mediaProjection - 屏幕投影：**
```xml
<service
    android:name=".ProjectionService"
    android:foregroundServiceType="mediaProjection"
    android:exported="false" />
```
需要`MEDIA_PROJECTION_SERVICE`权限。

## 7.1.10.7 多个类型

“如果我的应用同时需要多个类型怎么办？”洛FR问。

“好问题！”黛琳说道，“可以用'或'的方式组合。”

```xml
<!-- 音乐播放器需要同时用麦克风（K歌功能） -->
<service
    android:name=".MusicService"
    android:foregroundServiceType="mediaPlayback|microphone"
    android:exported="false" />
```

“但是，”黛琳补充道，“组合类型意味着需要满足所有类型的权限要求。比如你组合了`mediaPlayback|microphone`，那就需要同时有播放权限和麦克风权限。”

## 7.1.10.8 类型检查

“在Android 13上，系统还会检查类型是否匹配，”黛琳说道。

```kotlin
// 启动服务时，系统会检查
override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
    // 如果你在Manifest声明了camera类型
    // 但实际上启动的是location服务
    // 系统会抛出SecurityException!
    
    startForeground(NOTIFICATION_ID, notification)
    return START_STICKY
}
```

“所以一定要确保：声明的类型 = 实际做的类型，”黛琳强调道。

## 7.1.10.9 专业技术总结

本章我们学习了前台服务的各种类型。

**核心要点：**

1. **Android 12+引入了前台服务类型** - 需要在Manifest声明
2. **每种类型有对应的权限** - 不能混淆
3. **类型必须匹配** - 声明的类型必须和实际做的一致
4. **可以组合多个类型** - 用`|`连接
5. **权限是累加的** - 组合类型需要满足所有权限要求

**类型与权限对应：**

| 类型 | 特定权限 |
|------|---------|
| mediaPlayback | FOREGROUND_SERVICE_MEDIA_PLAYBACK |
| location | FOREGROUND_SERVICE_LOCATION + 位置权限 |
| camera | FOREGROUND_SERVICE_CAMERA + CAMERA |
| microphone | FOREGROUND_SERVICE_MICROPHONE + RECORD_AUDIO |
| phoneCall | ANSWER_PHONE_CALLS |
| dataSync | 无额外权限 |
| mediaProjection | MEDIA_PROJECTION_SERVICE |
