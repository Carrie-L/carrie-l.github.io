# 7.1.11 林间的休止符

上午的阳光透过森林的枝叶，在地面上投下斑驳的光影。露营编程旅团的姑娘们在森林中漫步，偶尔停下来采集一些树叶做标本。

“黛琳姐姐，”洛FR突然问道，“服务启动都说完了，那怎么停止呢？”

黛琳笑着摘下一片红叶：“停止前台服务也是有讲究的，咱们今天就来说说。”

## 7.1.11.1 停止的方式

“停止前台服务有三种方式，”黛琳扳着手指说道。

```kotlin
// 方式一：从外部停止
fun stopFromActivity() {
    // 使用stopService
    val intent = Intent(this, MusicService::class.java)
    stopService(intent)
    
    // 或者使用stopForegroundService (Android 8.0+)
    // 效果是一样的
}

// 方式二：从Service内部停止
class MusicService : Service() {
    
    fun stopPlayback() {
        // 停止前台服务（可以保留通知）
        stopForeground(STOP_FOREGROUND_DETACH)
        
        // 或者完全停止（移除通知）
        stopForeground(STOP_FOREGROUND_REMOVE)
        
        // 完全停止Service
        stopSelf()
    }
}

// 方式三：通知按钮停止
fun createStopAction(): NotificationCompat.Action {
    val intent = Intent(this, MusicService::class.java).apply {
        action = "com.example.ACTION_STOP"
    }
    
    val pendingIntent = PendingIntent.getService(
        this, 0, intent,
        PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
    )
    
    return NotificationCompat.Action.Builder(
        R.drawable.ic_stop, "停止", pendingIntent
    ).build()
}
```

## 7.1.11.2 stopForeground详解

“`stopForeground()`是停止前台服务的核心方法，”黛琳重点说道。

```kotlin
// stopForeground有两个参数
// STOP_FOREGROUND_DETACH - 停止前台服务，但保留通知
stopForeground(STOP_FOREGROUND_DETACH)

// STOP_FOREGROUND_REMOVE - 停止前台服务，同时移除通知
stopForeground(STOP_FOREGROUND_REMOVE)
```

“什么区别呢？”伊莎问。

“`DETACH`就像是把服务从'前台'状态脱离出来，但通知还在，”黛琳解释道，“用户还能看到通知，但Service已经不是前台服务了。”

“`REMOVE`更彻底，直接把通知也关掉，”她继续说道，“Service也完全停了。”

## 7.1.11.3 何时停止

“那什么时候应该停止呢？”洛FR问。

“有几种典型情况，”黛琳说道。

```kotlin
class MusicService : Service() {
    
    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        
        // 情况一：收到停止指令
        if (intent?.action == ACTION_STOP) {
            stopForeground(STOP_FOREGROUND_REMOVE)
            stopSelf()
            return START_NOT_STICKY
        }
        
        // 情况二：音乐播放完毕
        mediaPlayer.setOnCompletionListener {
            // 播放完毕，停止服务
            stopForeground(STOP_FOREGROUND_REMOVE)
            stopSelf()
        }
        
        // 情况三：用户切换到其他应用且不在后台播放
        if (!userWantsBackgroundPlay && !isAppInForeground()) {
            // 用户设置了不在后台播放
            stopForeground(STOP_FOREGROUND_REMOVE)
            stopSelf()
        }
        
        // 情况四：耳机断开连接
        val bluetoothReceiver = object : BroadcastReceiver() {
            override fun onReceive(context: Context, intent: Intent) {
                if (intent.action == BluetoothAdapter.ACTION_DISCONNECTION_TIMEOUT) {
                    // 耳机断开，停止播放
                    stopForeground(STOP_FOREGROUND_REMOVE)
                    stopSelf()
                }
            }
        }
        
        return START_STICKY
    }
}
```

## 7.1.11.4 清理资源

“停止的时候一定要清理资源，”黛琳严肃地说。

```kotlin
override fun onDestroy() {
    super.onDestroy()
    
    // 1. 停止媒体播放
    mediaPlayer?.apply {
        if (isPlaying) stop()
        release()
    }
    
    // 2. 释放其他资源
    unregisterReceiver(bluetoothReceiver)
    unregisterReceiver(headsetReceiver)
    
    // 3. 取消通知（如果还在的话）
    val notificationManager = getSystemService(NotificationManager::class.java)
    notificationManager.cancel(NOTIFICATION_ID)
    
    // 4. 保存播放进度（如果需要）
    savePlaybackProgress()
}
```

## 7.1.11.5 反模式

洛FR问：“有什么常见的错误吗？”

“有！”黛琳说道，“最常见的就是忘记清理资源。”

```
// 反模式：直接调用stopSelf不清理
override fun onDestroy() {
    // 错误！什么都没做
    super.onDestroy()
}
```

“这样会导致内存泄漏，”黛琳警告道，“MediaPlayer、Receiver等资源没有释放，会造成资源浪费。”

## 7.1.11.6 专业技术总结

本章我们学习了如何停止前台服务。

**核心要点：**

1. **三种停止方式** - stopService、stopForeground、stopSelf
2. **stopForeground参数** - DETACH保留通知，REMOVE移除通知
3. **停止时机** - 播放完毕、用户停止、应用退出等
4. **清理资源** - onDestroy中释放所有资源
5. **广播监听** - 耳机断开等情况要特殊处理

**清理清单：**
- MediaPlayer.release()
- unregisterReceiver()
- cancelNotification()
- 保存播放进度
