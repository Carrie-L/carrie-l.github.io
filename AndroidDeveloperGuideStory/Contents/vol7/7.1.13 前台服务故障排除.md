# 7.1.13 午后的调试时光

下午的阳光懒懒地照着，姑娘们有些犯困。希尔提议说：“咱们来聊聊前台服务的常见问题和解决方法吧！”

## 7.1.13.1 常见错误

“第一个常见错误：忘记调用startForeground()，”希尔说道。

```kotlin
// 错误！
override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
    // 忘记调用startForeground
    doSomeWork()
    return START_STICKY
}

// 正确
override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
    startForeground(NOTIFICATION_ID, notification)
    doSomeWork()
    return START_STICKY
}
```

“系统会在几秒内杀掉这个Service，”希尔补充道。

## 7.1.13.2 权限问题

“第二个常见问题：权限不对，”希尔继续说道。

```kotlin
// 错误：没声明权限
// Manifest中没有FOREGROUND_SERVICE权限

// 正确
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />

// Android 12+ 还需要特定类型
<uses-permission android:name="android.permission.FOREGROUND_SERVICE_MEDIA_PLAYBACK" />

// 错误：没动态申请
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
    requestPermissions(arrayOf(Manifest.permission.FOREGROUND_SERVICE_MEDIA_PLAYBACK), 1)
}
```

## 7.1.13.3 通知渠道

“第三个问题：通知渠道未创建，”希尔说道。

```kotlin
// 错误：Android 8.0+没创建渠道
// 直接调用startForeground会崩溃

// 正确
private fun createNotificationChannel() {
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
        val channel = NotificationChannel(
            CHANNEL_ID,
            "服务通知",
            NotificationManager.IMPORTANCE_LOW
        )
        manager.createNotificationChannel(channel)
    }
}
```

## 7.1.13.4 调试技巧

“怎么调试前台服务呢？”洛FR问。

希尔分享了几个技巧：

```kotlin
// 1. 查看Logcat
// 过滤 "ActivityManager" 可以看到服务启动/停止日志

// 2. 使用adb命令
adb shell dumpsys activity services

// 3. 查看通知
adb shell dumpsys notification

// 4. 测试权限
adb shell pm grant <package> android.permission.FOREGROUND_SERVICE
adb shell pm grant <package> android.permission.FOREGROUND_SERVICE_MEDIA_PLAYBACK
```

## 7.1.13.5 常见异常

“最后是常见异常列表，”希尔总结道。

| 异常 | 原因 | 解决方法 |
|------|------|---------|
| SecurityException | 缺少权限 | 添加Manifest权限+动态申请 |
| IllegalStateException | 未调用startForeground | 在onStartCommand中调用 |
| IllegalArgumentException | foregroundServiceType不匹配 | 检查Manifest声明的类型 |
| NotificationChannelException | 渠道未创建 | 调用createNotificationChannel |
