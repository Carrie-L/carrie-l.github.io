# 7.1.6 山坡上的宣言

上午的阳光温暖而不炙热，露营编程旅团的姑娘们找了一处向阳的山坡，铺开野餐垫，正式开始今天的编程学习。

洛FR摆弄着她的手机：“黛琳姐姐，我现在知道了前台服务很重要，但是具体怎么配置呢？感觉好复杂的样子。”

“别担心，”黛琳笑道，“其实核心就是两部分：Manifest声明和权限请求。我们一个一个来。”

## 7.1.6.1 Manifest声明

“首先，我们需要在AndroidManifest.xml里声明Service，”黛琳说道，“这一步是必须的。”

```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.example.myapp">

    <!-- 前台服务权限 -->
    <uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
    
    <!-- 特定类型权限 -->
    <uses-permission android:name="android.permission.FOREGROUND_SERVICE_MEDIA_PLAYBACK" />
    <uses-permission android:name="android.permission.FOREGROUND_SERVICE_LOCATION" />
    
    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:theme="@style/AppTheme">
        
        <activity
            android:name=".MainActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
        
        <!-- 声明前台服务 -->
        <service
            android:name=".MusicService"
            android:enabled="true"
            android:exported="false"
            android:foregroundServiceType="mediaPlayback" />
            
    </application>

</manifest>
```

“你们看这个`foregroundServiceType`属性，”黛琳重点强调道，“Android 12及以上必须指定类型。`mediaPlayback`就是音乐播放的类型。”

“如果没有声明这个属性会怎样？”伊莎问。

“在Android 12上会崩溃，”黛琳严肃地说，“`IllegalArgumentException`，直接闪退。”

## 7.1.6.2 常见的前台服务类型

黛琳列举了所有可用的类型：

```xml
<!-- 音乐播放 -->
android:foregroundServiceType="mediaPlayback"

<!-- 位置定位 -->
android:foregroundServiceType="location"

<!-- 摄像头 -->
android:foregroundServiceType="camera"

<!-- 麦克风 -->
android:foregroundServiceType="microphone"

<!-- 通话中 -->
android:foregroundServiceType="phoneCall"

<!-- 同步数据 -->
android:foregroundServiceType="dataSync -->

<!-- 投影 -->
android:foregroundServiceType="mediaProjection"

<!-- 多媒体 -->
android:foregroundServiceType="remoteMessaging"
```

“如果你的服务不属于以上任何类型，”黛琳补充道，“可以不用声明，或者用`none`。”

```xml
<!-- 不需要特定类型 -->
<service
    android:name=".MyService"
    android:foregroundServiceType="" />
    <!-- 或者不写这个属性 -->
```

## 7.1.6.3 动态请求权限

“Manifest声明好了，下一步就是动态请求权限，”黛琳说道，“特别是Android 13以后，有些权限需要用户手动同意。”

洛FR问：“那具体怎么请求？”

“和普通权限一样，用`requestPermissions()`，”黛琳说道。

```kotlin
class MainActivity : AppCompatActivity() {
    
    companion object {
        // 权限请求码
        private const val REQUEST_FOREGROUND_SERVICE = 1001
        private const val REQUEST_MEDIA_PLAYBACK = 1002
    }
    
    fun startMusicServiceWithPermission() {
        // 检查是否已有权限
        if (checkForegroundServicePermission()) {
            // 有权限，直接启动服务
            startMusicService()
        } else {
            // 没有权限，申请权限
            requestForegroundServicePermission()
        }
    }
    
    private fun checkForegroundServicePermission(): Boolean {
        return if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
            // Android 13+ 需要检查具体类型权限
            checkSelfPermission(Manifest.permission.FOREGROUND_SERVICE_MEDIA_PLAYBACK) == 
                PackageManager.PERMISSION_GRANTED
        } else if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.Q) {
            // Android 10-12 需要 FOREGROUND_SERVICE
            checkSelfPermission(Manifest.permission.FOREGROUND_SERVICE) ==
                PackageManager.PERMISSION_GRANTED
        } else {
            // Android 9 以下不需要运行时权限
            true
        }
    }
    
    private fun requestForegroundServicePermission() {
        val permissions = if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
            arrayOf(Manifest.permission.FOREGROUND_SERVICE_MEDIA_PLAYBACK)
        } else if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.Q) {
            arrayOf(Manifest.permission.FOREGROUND_SERVICE)
        } else {
            // 不需要请求
            return
        }
        
        requestPermissions(permissions, REQUEST_FOREGROUND_SERVICE)
    }
    
    override fun onRequestPermissionsResult(
        requestCode: Int,
        permissions: Array<out String>,
        grantResults: IntArray
    ) {
        super.onRequestPermissionsResult(requestCode, permissions, grantResults)
        
        when (requestCode) {
            REQUEST_FOREGROUND_SERVICE -> {
                if (grantResults.isNotEmpty() && 
                    grantResults[0] == PackageManager.PERMISSION_GRANTED) {
                    // 用户授权了，启动服务
                    startMusicService()
                } else {
                    // 用户拒绝了
                    showPermissionDeniedMessage()
                }
            }
        }
    }
    
    private fun startMusicService() {
        val intent = Intent(this, MusicService::class.java)
        startForegroundService(intent)
    }
    
    private fun showPermissionDeniedMessage() {
        Toast.makeText(this, "需要前台服务权限才能后台播放音乐", Toast.LENGTH_LONG).show()
    }
}
```

“你们看，”黛琳指着代码解释道，“核心逻辑就是：先检查权限，有就启动，没有就请求。请求完了根据结果决定下一步。”

## 7.1.6.4 权限组合

伊莎好奇地问：“如果我的应用既要播放音乐，又要获取位置呢？”

“那就需要同时请求多个权限，”黛琳说道。

```kotlin
// 组合多个权限
private fun requestMultiplePermissions() {
    val permissions = mutableListOf<String>()
    
    // 音乐播放权限
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
        permissions.add(Manifest.permission.FOREGROUND_SERVICE_MEDIA_PLAYBACK)
    } else if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.Q) {
        permissions.add(Manifest.permission.FOREGROUND_SERVICE)
    }
    
    // 位置权限（如果需要后台定位）
    if (needsBackgroundLocation) {
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.Q) {
            // Android 10+ 需要后台位置权限
            permissions.add(Manifest.permission.ACCESS_BACKGROUND_LOCATION)
        }
    }
    
    if (permissions.isNotEmpty()) {
        requestPermissions(permissions.toTypedArray(), REQUEST_MULTI_SERVICE)
    }
}
```

## 7.1.6.5 权限被拒绝的处理

洛FR问：“如果用户拒绝权限怎么办？”

“这是个很现实的问题，”黛琳认真地说，“绝对不能因为被拒绝就偷偷启动服务，那样会被Google下架的。”

她继续说道：“正确的做法是：告诉用户为什么需要这个权限，给用户选择。如果用户实在不同意，就提供'仅前台'的降级方案。”

```kotlin
// 权限被拒绝后的优雅处理
fun onPermissionDenied() {
    // 方法一：显示解释对话框
    if (shouldShowRequestPermissionRationale(Manifest.permission.FOREGROUND_SERVICE)) {
        AlertDialog.Builder(this)
            .setTitle("需要权限")
            .setMessage("后台播放音乐需要此权限，否则退出应用后音乐将停止。")
            .setPositiveButton("再试一次") { _, _ ->
                requestForegroundServicePermission()
            }
            .setNegativeButton("取消") { _, _ ->
                showAlternativeSolution()
            }
            .show()
    } else {
        // 用户选择了"不再询问"
        showAlternativeSolution()
    }
}

private fun showAlternativeSolution() {
    // 降级方案：提示用户只能在App前台时播放
    AlertDialog.Builder(this)
        .setTitle("替代方案")
        .setMessage("您可以在设置中手动开启权限，或者保持App在前台使用。")
        .setPositiveButton("打开设置") { _, _ ->
            openAppSettings()
        }
        .setNegativeButton("知道了")
        .show()
}

private fun openAppSettings() {
    Intent(Settings.ACTION_APPLICATION_DETAILS_SETTINGS).apply {
        data = Uri.fromParts("package", packageName, null)
        startActivity(this)
    }
}
```

“你们要记住，”黛琳强调道，“用户拒绝权限是他们的权利。我们能做的，就是尽可能地解释、提供替代方案，但绝对不能'偷跑'。”

## 7.1.6.6 专业技术总结

本章我们学习了前台服务的Manifest声明和权限请求。

**核心要点：**

1. **Manifest中声明Service** - 用`<service>`标签，`android:foregroundServiceType`指定类型
2. **声明所需权限** - `FOREGROUND_SERVICE` + 特定类型权限
3. **动态请求权限** - Android 9+ 需要运行时请求
4. **检查权限后再启动** - 先检查再启动，不要先斩后奏
5. **优雅处理拒绝** - 提供解释和替代方案，不能偷偷启动

**常见权限列表：**

| 类型 | Manifest权限 | 运行时权限 |
|------|-------------|-----------|
| 音乐 | FOREGROUND_SERVICE_MEDIA_PLAYBACK | Android 13+ 检查 |
| 位置 | FOREGROUND_SERVICE_LOCATION | + ACCESS_BACKGROUND_LOCATION |
| 摄像头 | FOREGROUND_SERVICE_CAMERA | Android 13+ 检查 |
| 麦克风 | FOREGROUND_SERVICE_MICROPHONE | Android 13+ 检查 |
