# 第三部分 **Kotlin 函数式编程基础入门目录**

## **3.1 思维转变：什么是函数式编程？**

* **3.1.1 函数是一等公民**
    * 理解函数可以像变量一样被传递和返回。
    * 示例：将函数存入变量，作为参数传入另一个函数。
* **3.1.2 拥抱不可变性 (Immutability)**
    * `val` 和 `var` 的核心区别与选择。
    * 不可变集合 (`listOf`, `setOf`, `mapOf`) 的好处：线程安全与可预测性。
* **3.1.3 纯函数与副作用 (Side Effects)**
    * 什么是纯函数？（给定相同输入，总有相同输出，且不影响外部环境）
    * 识别代码中的副作用（如打印、修改外部变量、读写文件）。
    * 为什么我们要尽量编写纯函数？

## **3.2 核心语法：Lambda 与高阶函数**

* **3.2.1 Lambda 表达式：代码块的艺术**
    * 语法：`{ 参数 -> 函数体 }`
    * 单个参数的隐式名称：`it`
    * 类型推断：Kotlin 如何自动识别 Lambda 类型。
* **3.2.2 函数类型：为函数“签名”**
    * 如何定义一个函数类型，例如 `(Int, Int) -> String`。
* **3.2.3 高阶函数：函数的“函数”**
    * 定义：接收函数作为参数，或返回一个函数的函数。
    * **动手实践**：
        * [ ] **Task 1**: 定义一个接收 `(Int) -> Int` 类型参数的高阶函数。
        * [ ] **Task 2**: 编写一个返回另一个函数的函数。
* **3.2.4 函数引用 `::`**
    * 如何直接引用一个已有的命名函数。

## **3.3 实战演练：强大的集合操作**

这是学习函数式编程最直观、最能获得成就感的部分。

* **3.3.1 转换 (`map`)**
    * 将集合中的每个元素映射成一个新的元素。
    * **动手实践**：
        * [ ] **Task 3**: 给一个 `List<Int>`，返回每个数字的平方组成的新列表。
* **3.3.2 过滤 (`filter`)**
    * 筛选出集合中所有满足条件的元素。
    * **动手实践**：
        * [ ] **Task 4**: 从一个 `List<Int>` 中，筛选出所有的偶数。
* **3.3.3 聚合 (`reduce`, `fold`)**
    * 将集合中的所有元素合并成一个单一的值。
    * `reduce` 和 `fold` 的区别（`fold` 有一个初始值）。
    * **动手实践**：
        * [ ] **Task 5**: 使用 `reduce` 计算一个 `List<Int>` 的总和。
* **3.3.4 其他常用操作**
    * `forEach`: 遍历元素（常用于执行副作用，如打印）。
    * `find`: 查找第一个满足条件的元素。
    * `groupBy`: 按条件将集合分组为 Map。
    * `flatMap`: 转换并“压平”集合。

## **3.4 标准库的馈赠：作用域函数**

这些函数能极大提升代码的流畅性和可读性，它们是高阶函数的绝佳应用。

* **3.4.1 `let`: 处理可空值**
    * 在非空对象上执行代码块，通常与安全调用符 `?.` 配合使用。
    * **动手实践**：
        * [ ] **Task 6**: 安全地处理一个可空的字符串，如果不为空则打印其长度。
* **3.4.2 `apply`: 对象配置**
    * 用于初始化或配置对象，返回对象本身，适合链式调用。
    * **动手实践**：
        * [ ] **Task 7**: 创建一个 `TextView` 对象并使用 `apply` 设置其 `text`, `textSize` 和 `textColor`。
* **3.4.3 `run`: 执行计算**
    * 与 `apply` 类似，但在 Lambda 中可以返回任何值。常用于在一个对象的上下文中执行一段计算。
    * **动手实践**：
        * [ ] **Task 8**: 对一个对象调用 `run`，执行一系列操作后返回一个结果。
* **3.4.4 `with`: 对象上下文操作**
    * `run` 的一个变体，但不是扩展函数。
    * **动手实践**：
        * [ ] **Task 9**: 使用 `with` 函数格式化一个包含用户信息的字符串。
* **3.4.5 如何选择它们？**
    * 根据上下文对象（`this` 还是 `it`）和返回值（对象本身还是 Lambda 结果）来区分和选择。


# **第四部分：Kotlin 高级函数式编程进阶目录**

## **第一章：深化函数式思想：抽象与组合**

* **4.1.1 函数组合 (Function Composition)**
    * 思想：将多个小函数像管道一样连接起来，构建复杂的功能。
    * 实践：实现 `compose` 和 `andThen` 帮助函数。
* **4.1.2 柯里化 (Currying) 与部分应用 (Partial Application)**
    * 思想：将一个接受多个参数的函数，转变为一系列接受单个参数的函数。
    * 目的：创建更专业、更易复用的函数。
    * 实践：编写一个 `curry` 函数，并应用于实际场景（如配置相似的组件）。
* **4.1.3 递归的艺术：超越循环**
    * 尾递归优化 (`tailrec`)：理解其原理，避免栈溢出，用递归实现高效的循环。
    * **核心实践**：使用 `tailrec` 重写复杂的循环逻辑（如斐波那契数列、集合操作）。
* **4.1.4 引用透明 (Referential Transparency)**
    * 深入理解纯函数的基石：一个表达式可以被其结果替换，而不改变程序的行为。
    * 这为何是并发编程和代码推理的圣杯。

#### **第二章：代数数据类型 (ADT)：构建健壮的业务模型**

* **4.2.1 和类型 (Sum Types)：模拟“或”关系**
    * 使用 `sealed class` / `sealed interface` 来创建严格的、有限的状态集。
    * **核心实践**：对网络请求结果建模：`sealed class NetworkResult { data class Success(...) object Loading; data class Error(...) }`
* **4.2.2 积类型 (Product Types)：模拟“与”关系**
    * `data class` 就是最典型的积类型。
    * 理解如何组合 `data class` 和 `sealed class` 来精确地描述业务领域。
* **4.2.3 函子 (Functor) 与 Monad 思想入门**
    * **函子**：任何“可被 `map`”的容器。理解 `map` 的本质是在不改变容器结构的前提下，转换其内部的值。(`List`, `Result`, `Flow` 都是函子)
    * **Monad**：任何“可被 `flatMap`”的容器。理解 `flatMap` (或 `mapNotNull`) 的作用：在转换的同时，还能改变容器的结构（如从 `List<List<Int>>` 变为 `List<Int>`）。这是处理嵌套上下文和链式操作的关键。

#### **第三章：核心模式：优雅地处理副作用**

* **4.3.1 告别 Null：`Option` 类型**
    * 思想：用 `Some(value)` 和 `None` 来显式地表示“值的存在与否”，彻底消灭 `NullPointerException`。
    * 实践：使用 Arrow 库的 `Option`，并对其进行 `map`, `flatMap`, `getOrElse` 等操作。
* **4.3.2 超越异常：`Either` 与 `Result`**
    * 思想：将成功 (`Right`/`Success`) 和失败 (`Left`/`Failure`) 两种可能性都作为函数的一等公民返回。
    * **核心实践**：重构一个会抛出异常的函数，使其返回 Arrow 的 `Either<ErrorType, SuccessType>`。学习如何通过 `map`/`flatMap` 在“成功”的路径上链式操作，而让错误自动传递。
* **4.3.3 累积错误：`Validated` 类型**
    * 场景：当需要一次性返回所有校验错误时（如表单验证），`Either` 只能返回第一个错误。
    * 实践：使用 Arrow 的 `Validated` 来实现累积多个错误的功能。

#### **第四章：Arrow Fx：纯函数式并发与资源管理**

* **4.4.1 理解“效应”(Effects)**
    * 思想：将副作用（网络请求、数据库读写、打印日志）封装在一个“效应”容器中（如 `IO` Monad），从而将“描述”一个操作与“执行”它分离开来。
* **4.4.2 `IO` Monad：延迟计算与副作用管理**
    * `IO { ... }` 只是构建了一个计算的蓝图，它是纯粹的、惰性的。
    * 只有在最后调用 `unsafeRunSync()` 或类似的函数时，副作用才会真正发生。
* **4.4.3 结构化并发与资源安全**
    * 使用 Arrow Fx 的 `parZip` / `parTraverse` 实现安全的并行计算。
    * `Resource` 类型：以纯函数的方式，优雅地管理需要打开和关闭的资源（如文件、数据库连接），告别 `try-finally`。
* **4.4.4 结合 Kotlin Coroutines**
    * 理解 Arrow Fx 如何与协程生态无缝集成，使用 `suspend` 函数构建 `IO`。

#### **第五章：函数式架构**

* **4.5.1 依赖注入的函数式视角**
    * Reader Monad：将依赖关系作为函数参数注入，而不是通过框架的魔法。这让单元测试变得极其简单。
* **4.5.2 分层架构中的函数式数据流**
    * 实践：设计一个应用（如 Repository -> UseCase -> ViewModel），其中每一层都返回 `Either` 或 `IO<Either<...>>`，实现一个端到端的、无异常的、类型安全的数据流。
* **4.5.3 函数式反应式编程 (FRP)**
    * 将 Kotlin Flow 视为一个处理异步事件流的函子/Monad。
    * 使用 `map`, `filter`, `flatMapConcat` 等操作符以函数式的方式组合和处理数据流。
