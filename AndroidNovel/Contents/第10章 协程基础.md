## 什么是协程？

“简单来说，协程就是**可暂停的代码块**。”小 Com 解释。

传统方式：
```kotlin
// 线程会阻塞，用户界面会卡住
Thread.sleep(1000)  // 等待1秒
val data = networkCall()  // 等待网络响应
```

协程方式：
```kotlin
// 不会阻塞，界面依然流畅
delay(1000)  // 协程暂停1秒
val data = withContext(Dispatchers.IO) {
    networkCall()  // 在后台线程执行
}
```

## 第一个协程

“先来看最简单的协程。”小 Com 写道：

```kotlin
import kotlinx.coroutines.delay
import kotlinx.coroutines.launch

// 在 Compose 里启动协程
@Composable
fun MyScreen() {
    var text by remember { mutableStateOf("开始") }
    
    Button(onClick = {
        // 启动协程
        CoroutineScope(Dispatchers.Main).launch {
            text = "等待中..."
            delay(1000)  // 暂停1秒
            text = "完成！"
        }
    }) {
        Text(text)
    }
}
```

## Launch vs Async

“协程有两种启动方式：”

| 方式 | 用途 | 返回值 |
|------|------|--------|
| `launch` | fire-and-forget | Job（可取消） |
| `async` | 需要返回值 | Deferred（可获取结果） |

```kotlin
// launch - 不需要返回值
CoroutineScope(Dispatchers.Main).launch {
    delay(1000)
    println("完成")
}

// async - 需要返回值
CoroutineScope(Dispatchers.Main).launch {
    val deferred = async { fetchUser() }
    val user = deferred.await()  // 等待结果
}
```

## Dispatchers：线程调度

“协程可以在不同线程执行。”小 Com 展示了 Dispatchers：

| Dispatcher | 用途 |
|------------|------|
| `Dispatchers.Main` | 主线程，UI 更新 |
| `Dispatchers.IO` | IO 操作（网络、数据库） |
| `Dispatchers.Default` | CPU 密集型（计算） |

```kotlin
CoroutineScope(Dispatchers.Main).launch {
    // UI 更新
    
    val data = withContext(Dispatchers.IO) {
        // 网络请求
        api.getData()
    }
    
    // UI 更新
    text = data
}
```

## ViewModel + 协程

“在 ViewModel 里用协程是最佳实践。”小 Com 展示了代码：

```kotlin
class MyViewModel : ViewModel() {
    
    private val _uiState = MutableStateFlow<UiState>(UiState.Loading)
    val uiState: StateFlow<UiState> = _uiState
    
    fun loadData() {
        viewModelScope.launch {
            _uiState.value = UiState.Loading
            
            try {
                val data = withContext(Dispatchers.IO) {
                    api.fetchData()
                }
                _uiState.value = UiState.Success(data)
            } catch (e: Exception) {
                _uiState.value = UiState.Error(e.message)
            }
        }
    }
}

sealed class UiState {
    object Loading : UiState()
    data class Success(val data: String) : UiState()
    data class Error(val message: String) : UiState()
}
```

## StateFlow + Compose

“用 StateFlow 配合 Compose，非常强大！”小 Com 说：

```kotlin
@Composable
fun MyScreen(viewModel: MyViewModel = viewModel()) {
    val uiState by viewModel.uiState.collectAsState()
    
    when (val state = uiState) {
        is UiState.Loading -> {
            CircularProgressIndicator()
        }
        is UiState.Success -> {
            Text(state.data)
        }
        is UiState.Error -> {
            Text(state.message, color = Color.Red)
        }
    }
}
```

## 实战：网络请求

“我们来做最后一个练习——网络请求！”小 Com 提议道。

```kotlin
// 1. 定义 API
interface ApiService {
    @GET("users/{id}")
    suspend fun getUser(@Path("id") id: String): User
}

data class User(val id: String, val name: String, val email: String)

// 2. Repository
class UserRepository {
    private val api = Retrofit.Builder()
        .baseUrl("https://api.example.com/")
        .build()
        .create(ApiService::class.java)
    
    suspend fun getUser(id: String): User {
        return api.getUser(id)
    }
}

// 3. ViewModel
class UserViewModel(private val repository: UserRepository) : ViewModel() {
    
    private val _user = MutableStateFlow<User?>(null)
    val user: StateFlow<User?> = _user
    
    private val _isLoading = MutableStateFlow(false)
    val isLoading: StateFlow<Boolean> = _isLoading
    
    fun loadUser(id: String) {
        viewModelScope.launch {
            _isLoading.value = true
            try {
                _user.value = repository.getUser(id)
            } catch (e: Exception) {
                _user.value = null
            } finally {
                _isLoading.value = false
            }
        }
    }
}

// 4. Compose 界面
@Composable
fun UserScreen(userId: String) {
    val viewModel: UserViewModel = viewModel()
    val user by viewModel.user.collectAsState()
    val isLoading by viewModel.isLoading.collectAsState()
    
    LaunchedEffect(userId) {
        viewModel.loadUser(userId)
    }
    
    Box(modifier = Modifier.fillMaxSize()) {
        when {
            isLoading -> {
                CircularProgressIndicator(
                    modifier = Modifier.align(Alignment.Center)
                )
            }
            user != null -> {
                Column(
                    modifier = Modifier.padding(16.dp)
                ) {
                    Text("ID: ${user!!.id}")
                    Text("姓名: ${user!!.name}")
                    Text("邮箱: ${user!!.email}")
                }
            }
            else -> {
                Text(
                    "加载失败",
                    color = Color.Red,
                    modifier = Modifier.align(Alignment.Center)
                )
            }
        }
    }
}
```

## 协程的取消

“协程是可以取消的。”小 Com 提醒：

```kotlin
val job = CoroutineScope(Dispatchers.Main).launch {
    try {
        while (isActive) {
            // 长时间运行的任务
            delay(100)
        }
    } catch (e: CancellationException) {
        println("协程被取消")
    }
}

// 取消协程
job.cancel()
```
