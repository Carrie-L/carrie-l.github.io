## 1. 打包类型

“Android App 有两种打包方式。”小 Com 介绍道：

| 类型 | 说明 | 用途 |
|------|------|------|
| **Debug APK** | 调试包 | 开发测试 |
| **Release APK** | 发布包 | 上架应用商店 |

---

## 2. 生成签名密钥

“首先需要签名密钥。”小 Com 展示了：

```bash
# 用 keytool 生成密钥
keytool -genkeypair -v -storetype PKCS12 -keystore my-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias

# 参数说明：
# -storetype PKCS12：密钥库类型
# -keystore my-key.jks：密钥库文件名
# -keyalg RSA：加密算法
# -keysize 2048：密钥长度
# -validity 10000：有效期（天）
# -alias my-key-alias：密钥别名
```

---

## 3. 配置签名

“在 build.gradle 中配置签名。”小 Com 展示了：

```kotlin
// build.gradle (app)
android {
    signingConfigs {
        create("release") {
            storeFile = file("my-key.jks")
            storePassword = "密码"
            keyAlias = "my-key-alias"
            keyPassword = "密钥密码"
        }
    }
    
    buildTypes {
        release {
            // 使用签名配置
            signingConfig = signingConfigs.release
            
            // 混淆
            isMinifyEnabled = true
            proguardFiles(
                getDefaultProguardFile("proguard-android-optimize.txt"),
                "proguard-rules.pro"
            )
        }
    }
}
```

---

## 4. ProGuard 混淆规则

“添加 ProGuard 混淆规则。”小 Com 展示了：

```kotlin
// proguard-rules.pro

# 保留 Kotlin Metadata
-keepattributes *Annotation*
-keepattributes Signature

# 保留数据类
-keep class com.example.app.data.** { *; }

# 保留 Retrofit
-keepattributes Exceptions
-keepclasseswithmembers class * {
    @retrofit2.http.* <methods>;
}

# 保留 Gson
-keep class com.google.gson.** { *; }
-keep class * implements com.google.gson.TypeAdapterFactory

# 保留 Room
-keep class * extends androidx.room.RoomDatabase

# 保留 Hilt
-keep class dagger.hilt.** { *; }

# 保留 Compose
-keep class androidx.compose.** { *; }
```

## 5. 构建 Release APK

“在 Android Studio 中构建 Release APK。”小 Com 展示了：

```bash
# 用命令行构建
./gradlew assembleRelease

# 构建完成后，APK 在
# app/build/outputs/apk/release/app-release.apk
```

---

## 6. 构建 Bundle (AAB)

“推荐构建 AAB（Android App Bundle）。”小 Com 展示了：

```kotlin
// build.gradle (app)
android {
    bundle {
        abi {
            enableSplit = true
            reset()
            include "armeabi-v7a", "arm64-v8a", "x86", "x86_64"
        }
        density {
            enableSplit = true
        }
        language {
            enableSplit = true
        }
    }
}
```

```bash
# 构建 AAB
./gradlew bundleRelease

# 生成的 AAB 在
# app/build/outputs/bundle/release/app-release.aab
```

## 8. Google Play 发布

“在 Google Play 发布 App。”小 Com 展示了步骤：

**1. 创建 Google Play 开发者账号**
- 费用：$25（一次性）

**2. 创建应用**
- 打开 Google Play Console
- 点击"创建应用"
- 填写应用名称

**3. 填写应用信息**
- 应用名称
- 简短描述
- 完整描述
- 应用图标
- 截图

**4. 上传 AAB/APK**
- 导航到"发布" > "测试"
- 上传 AAB/APK 文件

**5. 配置应用内容**
- 填写隐私政策
- 选择应用类型

**6. 发布**
- 点击"发布测试"或"发布正式版"

## 9. 版本管理

“正确管理版本号。”小 Com 展示了：

```kotlin
// build.gradle (app)
android {
    defaultConfig {
        versionCode 1          // 内部版本号，每次递增
        versionName "1.0.0"    // 用户看到的版本号
    }
}
```

**版本号规范：**
- `1.0.0` - 第一个正式版
- `1.0.1` - 小修复
- `1.1.0` - 新功能
- `2.0.0` - 大版本更新

## 10. 持续集成/部署 (CI/CD)

“用 GitHub Actions 自动构建。”小 Com 展示了：

```yaml
# .github/workflows/build.yml
name: Build APK

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up JDK
        uses: actions/setup-java@v3
        with:
          java-version: '17'
          
      - name: Grant execute permission
        run: chmod +x gradlew
        
      - name: Build Debug APK
        run: ./gradlew assembleDebug
        
      - name: Upload APK
        uses: actions/upload-artifact@v3
        with:
          name: app-debug.apk
          path: app/build/outputs/apk/debug/app-debug.apk
```
