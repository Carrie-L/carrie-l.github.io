## 什么是状态？

小 Com 带着她走进图书馆，找了一个舒适的角落坐下。

“我们先来想一个问题，”小 Com 说，“什么叫'状态'？”

林小满想了想：“状态……就是东西现在的样子？”

“差不多！”小 Com 打了个比方，“比如一个开关，它有两种状态：开（On）和关（Off）。一个输入框，它的状态就是里面的文字。一个计数器的状态，就是那个数字。”

“在 Compose 里，所有的 UI 都是由状态决定的。”小 Com 继续说，“状态变了，UI 就会自动跟着变。”

---

## 第一个状态：remember

小 Com 打开电脑，写下了第一段代码：

```kotlin
var count by remember { mutableStateOf(0) }

Text(text = "计数：$count")
Button(onClick = { count++ }) {
    Text("+1")
}
```

“`remember` 和 `mutableStateOf` 是 Compose 状态管理的基石。”小 Com 解释道：

- `mutableStateOf(0)`：创建一个值为 0 的状态
- `remember`：记住这个状态，不会在重组时重置
- `by`：委托语法，简化访问

---

## 状态改变 = UI 重建

“等等，”林小满问，“为什么 `count++` 会让 Text 更新？”

“这是 Compose 最神奇的地方！”小 Com 兴奋地说，“当状态改变时，Compose 会**自动重新执行**使用这个状态的代码块，这个过程叫'重组'（Recomposition）。”

```kotlin
// 伪代码解释
@Composable
fun Counter() {
    var count by remember { mutableStateOf(0) }  // 状态
    
    Text(text = "计数：$count")  // 依赖 count，会在 count 变化时重组
    Button(onClick = { count++ }) {  // 点击改变 count
        Text("+1")
    }
}
```

小 Com 画了一幅图：

```
用户点击 → 状态改变 → 自动重组 → UI 更新
```

---

## remember vs rememberSaveable

“那 `remember` 会在什么时候失效？”林小满问。

“`remember` 只在当前 Composable 在屏幕上时有效。”小 Com 说，“如果屏幕旋转或者进程被杀死了，`remember` 的值就没了。”

“如果想在屏幕旋转后还保存数据呢？”

“那就用 `rememberSaveable`！”小 Com 展示了代码：

```kotlin
var count by rememberSaveable { mutableStateOf(0) }
```

“`rememberSaveable` 会自动把状态保存到 Bundle 里，屏幕旋转后会自动恢复。”

---

## 状态提升：把状态放在哪里？

“问题来了，”小 Com 说，“状态应该放在哪里？”

她举了一个例子：

```kotlin
@Composable
fun MyScreen() {
    var name by remember { mutableStateOf("") }
    
    Column {
        TextField(
            value = name,
            onValueChange = { name = it }
        )
        HelloMessage(name = name)
    }
}

@Composable
fun HelloMessage(name: String) {
    Text(text = "你好，$name！")
}
```

“在这个例子里，`name` 状态在 `MyScreen` 里定义，然后传给 `HelloMessage`。”小 Com 解释道，“这叫'状态提升'（State Hoisting）。”

“为什么要在父组件里？”林小满问。

“因为 `HelloMessage` 可能被多个地方调用，”小 Com 说，“把状态放在父组件，可以让多个子组件共享。”

---

## ViewModel：复杂业务的状态管理

“如果页面很复杂，有很多状态该怎么办？”林小满问。

小 Com 说：“这就需要 ViewModel 了！”

```kotlin
// 定义 ViewModel
class CounterViewModel : ViewModel() {
    var count by mutableStateOf(0)
        private set  // 只能在 ViewModel 里修改
    
    fun increment() {
        count++
    }
    
    fun decrement() {
        count--
    }
}

// 在 Compose 里使用
@Composable
fun CounterScreen(viewModel: CounterViewModel = viewModel()) {
    Column {
        Text(text = "计数：${viewModel.count}")
        Button(onClick = { viewModel.increment() }) {
            Text("+1")
        }
        Button(onClick = { viewModel.decrement() }) {
            Text("-1")
        }
    }
}
```

“为什么要在 ViewModel 里？”林小满问。

“好处很多！”小 Com 列举：

| 好处 | 说明 |
|------|------|
| 生命周期感知 | 状态在屏幕旋转时不丢失 |
| 状态共享 | 多个 Composable 可以共享 |
| 业务逻辑分离 | UI 和逻辑分开 |
| 更容易测试 | 逻辑可以单独测试 |

---

## StateFlow vs Compose State

“还有一种常见的写法，用 `StateFlow`。”小 Com 说：

```kotlin
class CounterViewModel : ViewModel() {
    private val _count = MutableStateFlow(0)
    val count: StateFlow<Int> = _count
    
    fun increment() {
        _count.value++
    }
}

@Composable
fun CounterScreen(viewModel: CounterViewModel = viewModel()) {
    val count by viewModel.count.collectAsState()
    
    Text(text = "计数：$count")
}
```

“什么时候用 `StateFlow`？”林小满问。

“简单界面用 Compose State，复杂界面用 StateFlow。”小 Com 建议道，“StateFlow 可以和协程配合，做网络请求等异步操作。”

## 实战：用状态做一个开关灯效果

“我们来做个好玩的！”小 Com 提议道。

```kotlin
var isOn by remember { mutableStateOf(false) }

Column(
    modifier = Modifier
        .fillMaxSize()
        .background(
            if (isOn) Color.Yellow else Color.DarkGray
        ),
    horizontalAlignment = Alignment.CenterHorizontally,
    verticalArrangement = Arrangement.Center
) {
    Icon(
        imageVector = if (isOn) Icons.Default.LightMode else Icons.Default.DarkMode,
        contentDescription = null,
        modifier = Modifier.size(100.dp),
        tint = if (isOn) Color.White else Color.Gray
    )
    
    Spacer(modifier = Modifier.height(32.dp))
    
    Text(
        text = if (isOn) "灯亮了！" else "灯关了",
        color = Color.White,
        style = MaterialTheme.typography.headlineMedium
    )
    
    Spacer(modifier = Modifier.height(32.dp))
    
    Button(
        onClick = { isOn = !isOn }
    ) {
        Text(if (isOn) "关灯" else "开灯")
    }
}
```

“太酷了！”林小满喊道，“点击按钮，背景颜色和图标都会变！”

## 本课小结

今天林小满学到了：

1. **状态（State）**：UI 的数据来源
2. **remember**：记住状态，不重组不重置
3. **mutableStateOf**：创建可观察的状态
4. **重组（Recomposition）**：状态变化自动更新 UI
5. **状态提升**：把状态放在合适的层级
6. **ViewModel**：复杂业务的状态管理
7. **StateFlow**：配合协程的异步状态流
