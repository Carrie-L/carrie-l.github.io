## 1. 协程 vs 线程

“先理解协程和线程的区别。”小 Com 介绍了：

| 特性 | 线程 | 协程 |
|------|------|------|
| 创建成本 | 高（需要 OS 支持） | 极低 |
| 切换成本 | 高（需要内核切换） | 极低（用户态） |
| 数量限制 | 有限（几百个） | 数十万 |
| 管理方式 | 操作系统 | 程序员控制 |

**关键点**：协程运行在线程内，但一个线程可以运行数十万个协程！

## 2. 协程的挂起

“先理解挂起（Suspension）。”小 Com 展示了：

```kotlin
// 这个函数会挂起
suspend fun fetchData(): String {
    // 模拟网络请求 - 这里会挂起
    delay(1000)  
    return "数据"
}
```

**挂起原理**：
- 挂起时不阻塞线程
- 线程可以去做其他事情
- 完成后自动恢复

## 3. 状态机

“协程本质是一个状态机！”小 Com 展示了：

```kotlin
// 源代码
suspend fun example() {
    val a = step1()    // 状态1
    val b = step2(a)  // 状态2
    val c = step3(b)  // 状态3
    println(c)
}

// 编译后变成（伪代码）
class ExampleContinuation : Continuation<String> {
    var label = 0
    var a: String? = null
    var b: String? = null
    
    override fun resumeWith(result: String) {
        when (label) {
            0 -> {
                // 开始 step1
                label = 1
                step1(this)
            }
            1 -> {
                // step1 完成，继续 step2
                a = result as String
                label = 2
                step2(a!!, this)
            }
            // ...
        }
    }
}
```

interface Continuation<in T> {

## 6. Dispatchers

“Dispatchers 决定协程在哪里运行。”小 Com 展示了：

```kotlin
// Dispatchers.Main - 主线程（UI 线程）
Dispatchers.Main

// Dispatchers.IO - IO 操作（网络、数据库）
Dispatchers.IO

// Dispatchers.Default - CPU 密集型
Dispatchers.Default

// Dispatchers.Unconfined - 不限制线程
Dispatchers.Unconfined
```

**实现原理**：
- 每个 Dispatcher 有一个线程池
- 协程被调度到线程池执行
- 完成后可以切换回原线程

## 8. 取消的实现

“协程取消是如何工作的？”小 Com 展示了：

```kotlin
suspend fun cancellableTask() {
    // 检查取消状态
    ensureActive()  // 如果已取消，抛出 CancellationException
    
    // 或者用 isActive
    if (!coroutineContext.isActive) {
        return
    }
    
    // 长时间操作
    for (i in 1..1000) {
        processItem(i)
        yield()  // 定期让出 CPU
    }
}

// 取消时
job.cancel()  // 发送取消信号
// 所有 ensureActive() 检查点都会抛出异常
```

## 9. 异常处理

“协程异常处理机制。”小 Com 展示了：

```kotlin
// 1. try-catch
suspend fun test() {
    try {
        riskyOperation()
    } catch (e: Exception) {
        println("捕获异常: $e")
    }
}

// 2. CoroutineExceptionHandler
val handler = CoroutineExceptionHandler { _, exception ->
    println("协程异常: $exception")
}

val scope = CoroutineScope(SupervisorJob() + handler)

// 3. 作用域异常传播
val scope = CoroutineScope(SupervisorJob())
scope.launch {
    throw Exception("错误")  // 不会影响其他协程
}
```

## 10. 实战：手写简化协程

“我们来实现一个简化版协程！”小 Com 提议道。

```kotlin
// 简化版协程框架

class SimpleCoroutine(
    private val block: suspend () -> Unit
) {
    private val continuation = object : Continuation<Unit> {
        override val context: CoroutineContext = EmptyCoroutineContext
        
        override fun resumeWith(result: Result<Unit>) {
            result.getOrNull()?.let { }
            result.exceptionOrNull()?.let { throw it }
        }
    }
    
    fun start() {
        block.startCoroutine(continuation)
    }
}

fun SimpleScope(
    block: suspend () -> Unit
): SimpleCoroutine {
    return SimpleCoroutine(block)
}

// 使用
fun main() {
    SimpleScope {
        println("开始")
        delay(100)  // 模拟挂起
        println("完成")
    }.start()
}
```
