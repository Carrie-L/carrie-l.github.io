## Android Choreographer & VSync 渲染机制

**WHAT**

Choreographer 是 Android UI 渲染的"指挥家"，负责将动画、输入事件、绘制任务统一调度到 VSync 信号节拍上执行，确保每一帧都在正确的时间点完成。

**WHY**

60Hz 屏幕每 16.67ms 发出一次 VSync 信号。如果主线程在这 16ms 内没有完成 measure / layout / draw 三个阶段，就会错过 VSync 节拍，导致当前帧被跳过——也就是常说的"掉帧"和"卡顿"。

**HOW**

1. **监听帧回调检测卡顿**：
   ```kotlin
   Choreographer.getInstance().postFrameCallback { frameTimeNanos ->
       val diff = (frameTimeNanos - lastFrameNanos) / 1_000_000
       if (diff > 16) Log.w("Perf", "掉帧！帧间隔: ${diff}ms")
       lastFrameNanos = frameTimeNanos
   }
   ```

2. **Perfetto / Systrace 分析**：抓取 `doFrame` 各阶段耗时，定位是 input、animation 还是 draw 阶段超时。

3. **常见优化方向**：
   - 减少过度绘制（开发者选项 → 显示过度绘制区域）
   - RecyclerView 多级缓存 + DiffUtil 异步 diff
   - `ViewStub` 懒加载不常用视图
   - `AsyncLayoutInflater` 异步 inflate 复杂布局
