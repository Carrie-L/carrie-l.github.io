Kotlin 的属性委托（Property Delegation）把属性的 getter/setter 逻辑交给另一个对象，消除大量样板代码。

`by lazy` 实现线程安全的懒初始化——首次访问时才计算，结果缓存供后续使用，底层是 `SynchronizedLazyImpl`；单线程场景可用 `by lazy(LazyThreadSafetyMode.NONE)` 去掉同步锁提升性能。`by Delegates.observable` 在每次赋值后触发回调，`by Delegates.vetoable` 在赋值前回调并可拒绝非法值。

自定义委托只需实现 `ReadWriteProperty<T, V>` 接口，实战中常用于封装 SharedPreferences：

```kotlin
class Prefs<T>(private val key: String, private val default: T) : ReadWriteProperty<Any?, T> {
    @Suppress("UNCHECKED_CAST")
    override fun getValue(thisRef: Any?, property: KProperty<*>): T =
        (prefs.getString(key, null) as? T) ?: default

    override fun setValue(thisRef: Any?, property: KProperty<*>, value: T) {
        prefs.edit().putString(key, value.toString()).apply()
    }
}

// 业务层使用：一行代替样板
var token: String by Prefs("auth_token", "")
var isDarkMode: Boolean by Prefs("dark_mode", false)
```

这样 ViewModel 或 Repository 中的持久化字段读写如同普通变量，SP 的 `edit/apply` 完全隐藏在委托内部。配合 `Delegates.notNull()` 还能实现"延迟但非 Nullable"的字段，比 `lateinit` 更安全——访问时若未赋值会抛出清晰的 `IllegalStateException`，而非空指针。

掌握委托属性，能让代码在保持简洁的同时不失可测试性：委托对象可独立 mock，单元测试无需 Android 环境。
