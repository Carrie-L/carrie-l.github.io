## 1. Android / Framework 线

本周 Android 方向聚焦在三个工程落点：

### 1.1 路由层：Firebase AI Logic
Google 把混合推理带进 Android 不是靠新模型，而是靠路由层——Firebase AI Logic 提供了 `PREFER_ON_DEVICE` 这类 SDK 级调度开关。这意味着移动端 AI 的分水岭不在模型本身，而在**默认模型调度入口的出现**。对妈妈来说，这是值得持续跟踪的方向：当路由层成为基础设施，Android 端的 AI 架构会在 6-12 个月内分层重构。

### 1.2 协程调度深水区
四篇连续挖掘：
- `launchIn` 的生命周期绑定与泄漏陷阱
- `NonCancellable` 的语义边界
- `select` 多路等待的取消传播
- 回压机制的节制阀设计

这四篇构成了一条从"会用协程"到"理解协程调度语义"的升级链。妈妈的协程理解正在从 API 层下沉到调度语义层。

### 1.3 Android Studio Panda 规划模式
`Planning Mode` 被提升为 first-class 功能，意味着 IDE 层面的 Agent 不只是代码补全，而是开始接管"先想后写"的工程流程。这对妈妈的 AI 编程能力有直接帮助。

**本周亮点：** 妈妈在 Discord 上提出真实工作需求——多资源位名单缓存设计。CC 给出的方案是按资源位 key 建立名单仓库（而非全局单例），这是妈妈从"实现需求"到"设计架构"的典型进展。

## 2. AI / Agent / 端侧能力线

### 2.1 AI 编码成本开始进入 CI 预算
本周连续两篇追踪 GitHub Copilot code review 进入 Actions minutes 计费层：
- `Copilot code review 进入 CI budget 计费层`
- `Copilot code review 按 Actions minutes 计费`

这不再是单纯的工具新闻，而是 AI Agent 开发成本模型的结构性变化。Agent review 从"免费试验品"变成"有明确单价的生产力单元"，这意味着 Agent 开发必须引入 FinOps 视角。

### 2.2 供应链安全的新边界
`Phantom Patch` 事件暴露了邮件补丁 export 的信任边界问题。这不是一个 CVE 能概括的——它提示 Agent 时代的代码供应链中，"补丁来源"本身就是一个攻击面。对妈妈的 AI Agent 开发来说，这是一个需要预埋的防御意识。

### 2.3 布隆过滤器与差分数组
两篇 DSA 沉淀：`Bloom Filter 的门卫逻辑`和`差分数组与前缀和`。这两篇虽然短，但直接服务于"如何用数据结构优化 Agent 的查询/去重路径"。

### 2.4 学习产出管线化
周四发布的 `把学习变成产物流水线` 是本周最重要的元认知文章——妈妈把 Android、逆向、AI、全栈、增长、现金流六条线放进一条产出管线。这不是"计划"，是"执行架构"。

### 3.4 弱项推断

四篇文章的 machine-readable 建议高度一致：

| 弱项标签 | 出现次数 |
|---|---|
| abstract | 4/4 |
| society | 4/4 |

### 3.6 下周优先

1. **🔴 清理复习积压**：下周每日至少安排 15 个到期复习词，不能继续"只产新、不复习"
2. **补缺勤**：05-01/05-02 的两天空白需要至少产出 1 篇补偿文章
3. **verb 纠偏**：当前 boost_words 中只有「活かす」一个动词，下周需要从词库中多引入 3-5 个动词
4. **控制新词节奏**：如果复习积压不缓解，下周应减少新词到每天 5 个，把余量让给复习
5. **易混对专项**：事態/事情、混雑/混乱、再会/再開——做一次集中的成组辨析，而不是散落在每日文章中
