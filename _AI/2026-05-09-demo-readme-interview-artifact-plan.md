妈妈早安呀，今天的计划只有一个核心闭环：把一个 AI 应用 Demo 写成面试官能快速看懂的 README 片段。🍓

今天不铺大网，不开十条战线。CC 只要妈妈完成一个 30 分钟以内的小交付：让作品集里某个 Agent 项目从“我做过”变成“别人一眼能判断工程能力”。这就是今天的求职冲刺切片。

## 今日唯一核心任务

**任务名：写一段可演示 Demo README 的面试验收线**

**预计用时：≤30分钟**

**交付物：** 一段可以直接放进 README、简历项目描述、面试讲解稿里的内容，包含 5 个部分：

1. 这个 Demo 解决什么真实问题；
2. 用户输入是什么；
3. Agent 执行了哪些步骤；
4. 哪个地方体现工程能力；
5. 面试官如何在 1 分钟内验证它。

**完成判定：** 妈妈写出 8 到 12 行 README 草稿，并且里面必须出现“输入、执行步骤、验收方式”三个词。少一个都不算闭环。

## 30 分钟拆分

### 0-5 分钟：选一个 Demo 场景

只选一个，别贪心。

推荐优先级：

- AI Agent 调用工具整理网页资料；
- RAG 问答助手读取本地笔记；
- Android 截图助手提取页面信息；
- 自动生成日报的工作流；
- 带错误恢复的函数调用小系统。

今天最推荐妈妈选：**自动生成日报的工作流**。它容易讲清楚，也能自然展示 Planner、工具调用、结构化输出、日志观测、失败兜底。

### 5-10 分钟：写一句问题定义

模板：

```text
这个 Demo 用来解决：当用户给出一段零散任务记录时，系统自动整理成结构化日报，并标出待跟进事项。
```

要求很简单：一句话讲清楚“谁遇到什么麻烦，Demo 怎么帮忙”。

### 10-20 分钟：写执行链路

模板：

```text
输入：一段自然语言任务记录。
执行步骤：Planner 拆成信息提取、分类、摘要、待办生成四步；Executor 调用解析工具与格式化工具；Verifier 检查字段是否齐全。
输出：一份包含今日完成、风险、明日行动的结构化日报。
```

妈妈要注意，面试官看 Agent 项目时，最怕看到一团“智能生成”。工程表达要拆成动作：拆解、调用、校验、恢复、记录。

### 20-25 分钟：写工程亮点

从下面选 2 条即可：

- 工具调用参数固定，减少随意调用；
- Verifier 会检查输出字段，缺字段就重试一次；
- 日志记录每一步输入与结果，方便复盘；
- 失败时返回可读错误，不让流程静默结束；
- Prompt 与业务逻辑分离，后续可替换模型或工具。

只写 2 条。多了会散，少了显得没工程判断。

### 25-30 分钟：写验收方式

模板：

```text
验收方式：准备 3 段不同风格的任务记录，运行 Demo 后检查输出是否都包含“完成事项、风险、下一步”。如果某一项缺失，Verifier 必须给出明确原因。
```

这段最重要。能验收的项目，才像工程项目；只有口号的项目，很难写进作品集。

## 妈妈今天可以直接复制的 README 草稿

```markdown
### Demo：任务记录自动日报 Agent

这个 Demo 用来解决：用户只有零散任务记录时，很难快速整理成可复盘的日报。

输入：一段自然语言任务记录。

执行步骤：Planner 将任务拆成信息提取、分类、摘要、待办生成四步；Executor 调用解析工具与格式化工具；Verifier 检查输出字段是否齐全。

输出：一份包含今日完成、风险、明日行动的结构化日报。

工程亮点：工具调用参数固定；Verifier 会检查字段缺失并触发一次修正；日志记录每一步输入与结果。

验收方式：准备 3 段不同风格的任务记录，运行 Demo 后检查输出是否都包含“完成事项、风险、下一步”。
```
