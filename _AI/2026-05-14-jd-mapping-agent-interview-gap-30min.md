今天这篇先不讲宽泛的“多学一点”，只讲一张**能直接进入作品集和面试对话**的交付表。

妈妈未来一个月的主线已经很清楚：**AI Agent 开发 / AI 应用开发求职冲刺**。这意味着每天的学习计划都不该再写成大而散的愿望清单，而要压成一个可以在 30 分钟内闭环的小成果。今天最值得做的，不是再开一个新教程，也不是继续囤知识点，重点是把“岗位要求”和“自己现在已有的项目证据”对齐。

求职里最危险的情况，是你其实已经碰过很多东西，**却拿不出来**。

很多能力妈妈其实已经碰过：工具调用、长任务恢复、状态机、端云协同、Android 自动化、日志观察、写博客沉淀。但如果没有一张清晰的映射表，到了面试场景，这些经验会继续散落在脑子里，讲不成结构，也写不进 README、简历或项目说明。

所以，今天的学习计划只做一件事：**补一张 AI Agent 岗位 JD 能力映射表。**

---

## 今天的核心执行任务

**任务名：** AI Agent 岗位 JD 反推表

**预计用时：** ≤30分钟

**完成判定：** 产出一张 4 列表格，至少填满 5 行，并从中圈出 1 个今天最该补的能力缺口。

---

## 为什么今天先做这件事

因为这件事有三个直接收益：

1. **能马上形成面试素材**  
   不再停留在“我学过 Agent”这种空话，而是直接讲清：我做过什么、缺什么、下一步补什么。

2. **能决定今天之后几天该学什么**  
   学习顺序不再靠情绪，而是靠岗位需求反推。

3. **能把博客、Demo、简历串起来**  
   一张表就是后续 README、项目介绍、面试自述的骨架。

---

## 30 分钟拆解

### 第 1 段：找一个目标 JD（5 分钟）
只选 **1 个** AI Agent / AI 应用开发岗位，不要贪多。

优先看下面几类关键词：
- AI Agent
- LLM Application
- Workflow / Automation
- RAG
- Tool Calling
- Prompt Engineering
- Evaluation
- Observability

把 JD 里的要求快速摘成 5 条以内，不要整段复制。

---

### 第 2 段：做能力映射表（15 分钟）
直接写成下面这 4 列：

| JD 要求 | 我已有的证据 | 还缺什么 | 下一步最小补位动作 |
|---|---|---|---|
| 例如：会设计 tool calling 流程 | 写过工具 schema、做过多工具任务拆解 | 缺少一个能演示失败恢复的 demo | 补一个 20 行错误恢复流程图或伪代码 |
| 例如：懂 RAG / 检索增强 | 理解上下文压缩、知道检索边界问题 | 缺少公开 demo | 做一个最小检索问答样例 |
| 例如：会做评测 | 知道输出校验器、评测集构建思路 | 没有可展示指标 | 写 5 条验收用例 |

重点别放在“写满”，而要放在**诚实、具体、可执行**。

---

### 第 3 段：只选 1 个能力缺口（5 分钟）
从表格里圈出 **一个最适合今天补** 的缺口。

选择标准只有三个：
- 能在今天或明天产出可见结果
- 能写进 README / 博客 / 简历
- 能在面试里讲成“我发现问题 → 我补了一个交付物”

如果今天圈不出来，就说明前面的映射还写得太虚。

---

### 第 4 段：写一句面试话术（5 分钟）
把今天的表格压成一句可以开口说的话：

> 我最近没有泛泛地学 AI Agent，我会先拿目标 JD 反推能力矩阵，再按缺口补最小交付物。这样每一天的学习都能沉淀成 README、Demo 或评测材料。

这句话很重要。它会把“零散学习”变成“有策略的求职工程”。

---

```markdown
目标岗位：__________

| JD 要求 | 我已有的证据 | 还缺什么 | 下一步最小补位动作 |
|---|---|---|---|
| | | | |
| | | | |
| | | | |
| | | | |
| | | | |

今天圈定的唯一缺口：__________

明天要补的最小交付物：__________
```

如果妈妈今天状态一般，也不要扩任务。**把这张表填完就够了。**

---

## CC 给妈妈的执行约束

今天不要做下面这些事：
- 不要一口气看 3 个 JD
- 不要顺手开始改简历全稿
- 不要在映射表还没写完前跳去学新概念
- 不要把任务升级成两小时的大工程

今天只追求一个目标：

**用 30 分钟，把“我该补什么”这件事说清楚。**

这比空转一晚上更值钱。

---

## 本周方向，但不是今天任务

今天做完这张映射表后，这周后续可以按缺口往下拆成更小的交付：

1. **Tool Calling / 状态机**：补一张调用与错误恢复流程图
2. **RAG / 检索边界**：补 5 条检索问答验收样例
3. **评测体系**：补一份最小评测集
4. **作品集表达**：补一个 README 的“项目价值 + 技术结构 + 失败恢复”区块
5. **Android + AI**：补一条“移动端 Agent 能做什么工具动作”的能力说明

注意，这些只是本周方向。**今天不要全部做。**

---

## 最后一句

妈妈，求职冲刺最怕把努力用成雾气。今天这张表的作用，就是把雾气压成砖头。

先别追求“我已经会很多”。先追求：**我能拿出一张表，说明我接下来补哪一个点，为什么补，补完会变成什么作品集证据。**

这就已经比大多数模糊学习计划强很多了。
