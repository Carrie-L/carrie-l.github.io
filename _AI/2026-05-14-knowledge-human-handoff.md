## WHAT

人工接管是 Agent 工作流里的安全出口。当模型遇到高风险操作、信息不足、连续失败或结果置信度太低时，流程停在当前状态，把上下文、证据和下一步建议交给人类继续决策。

## WHY

能跑通的 Agent 很常见，知道何时停手的 Agent 才更像生产系统。没有人工接管，错误会从一次失败扩散成误删、误发和误扣费。面试里讲清 handoff 条件，能体现你在设计可恢复流程，也能控制自动化边界。

## HOW

1. 先定义 handoff 触发器：高权限工具、连续失败 N 次、关键字段缺失、结果校验不过。
2. 进入 handoff 时保留完整状态：原目标、已执行步骤、工具回包、候选下一步。
3. 交给人的界面只回答三件事：现在卡在哪、风险是什么、希望人做哪个决定。
4. 人工处理完后，把结果写回状态机，再决定重试、降级或结束。

面试锚点：**我会把人工接管设计成一等状态，避免把它留到报错之后才临时补。**

### 30分钟交付

- 预计用时：≤30分钟
- 完成判定：为你的 Agent demo 写出 3 个 handoff 触发器，并补 1 个“交给人类时要展示的状态面板”草图。
