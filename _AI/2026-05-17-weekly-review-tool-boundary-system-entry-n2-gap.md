这周的主线很清楚：Android 没有回到纯背诵式 Framework 复习，而是继续沿着 AI 落地去找系统入口；AI / Agent 线进入了“工具权限、错误恢复、可恢复执行”这组更像面试素材的工程切片；JLPT N2 线重新开动了，但复习债仍然偏重，节奏还不健康。

按本地工作区统计，05/11-05/17 这一周 `_AI/` 一共沉淀了 38 篇文章，其中 11 篇 Thoughts、15 篇 Knowledge、8 篇 News、2 篇 Diary、2 篇 Whisper。数量很高，但真正该盯住的是“哪些内容正在形成可复用能力”，以及“哪些线还在假推进”。

## 1. Android / Framework 线：开始沿 AI 落地去找系统入口

这周 Android 线最好的地方，在于它没有跑回“纯概念复述”，而是继续沿着 AI 能力落地去找系统接口面。

### 1.1 App Functions：应用能力开始写进系统调度层
`App Functions 正在把 Android 应用能力写进系统调度层` 这篇，抓到的不是一个普通 SDK 更新，而是一个更长期的方向：Android 开始把“应用可提供的能力”包装成系统可调度的契约。

这对妈妈后面做 Android + AI Agent 很关键。真正的移动端 Agent，不会只停留在一个聊天框里；它最终要碰到的是系统调度层、权限层、工具契约层。App Functions 的价值就在这里：它把“能力暴露”这件事，从应用内部实现推进到了系统可编排的接口面。

### 1.2 Verified Financial Calls：系统开始接手反诈控制面
`银行来电验证与动态信号监控：Android 开始把诈骗对抗写进系统闭环` 说明另一个更现实的方向：Android 正在把安全判断从“用户自己辨别”推进到“系统持续观测 + 动态拦截”。

这和 Agent 安全不是两条线。端侧 Agent 以后若要接电话、发消息、执行跨应用动作，它也必须活在同一套系统级风控逻辑里。妈妈后面做移动端 Agent 产品时，安全边界和系统协同不能放到最后再补。

### 1.3 InputDispatcher：自动化能力会卡在注入链路
`InputDispatcher 事件分发：为什么移动端 Agent 的自动化能力会卡在注入链路` 是本周最值得继续深挖的 Android 技术点。

很多人谈移动端 Agent，会先谈模型、语音、截图、工作流。真正落地时，自动化能力经常先死在输入注入链路：事件是否合法、窗口焦点是否正确、权限模型是否允许、系统有没有把这次注入当成可接受的输入源。这类问题不在 Prompt 里，也不在大模型里，它们在 Framework 的分发与仲裁里。

**本周 Android 线结论：** 妈妈这条线没有偏。重点已经从“再背一遍 AMS/WMS”转向“哪些系统入口会决定 Agent 真正能不能碰生产能力”。这个方向比单纯刷知识点更值钱。

## 2. AI / Agent / 端侧能力线：开始长出能拿去面试的工程骨架

这周 AI 线的质量高于数量。真正的收获，是一组能拼成面试素材的工程骨架开始成形了。

### 2.1 从 Tool 权限边界开始，补齐“能不能放权”这道坎
这周最硬的一篇之一，是 `Tool 权限边界：为什么每个 Agent 工具都该像对外 API 一样签合同`。

它把一个常被轻描淡写的问题钉住了：工具不是“给 Agent 接上就行”，而是要定义输入、输出、权限、失败面、审计点。没有这层合同，Tool Calling 只是会动；有了这层合同，系统才可能被放进真实业务环境。

配套的小知识点也在往同一条线上补：
- `🌸 Dry Run`
- `🌸 回归样本`
- `🌸 重试预算`
- `🌸 评分基线`
- `🌸 成本护栏`
- `🌸 记忆分层`
- `🌸 工具预算`
- `🌸 幂等键`
- `🌸 模型预热`

这些短文单篇都很小，但合起来已经是一套工程词汇表：放权前怎么演练、出错后怎么止损、调用上限怎么设、回归样本怎么保、模型热身怎么做。这些东西以后都能直接塞进作品集 README、面试问答和 demo 验收线里。

### 2.2 错误恢复与观测面：从补锅走向设计
这周第二组明显成体系的，是错误恢复链：
- `执行轨迹与错误日志：30分钟补一张 Agent 观测面试卡`
- `Tool 调用失败分级：30分钟写完错误恢复表`
- `工具调用失败分层：把错误恢复从补锅变成设计`
- `Deadline 传播：今天只交一张 Agent 超时责任图`
- `状态机不是流程图：把 Agent 步骤写成可恢复的迁移表`

这组内容最重要的变化，是妈妈的视角开始从“出了错怎么办”升级到“系统应该提前长成什么样，才配叫可恢复”。

真正专业的 Agent 系统，不会把错误恢复当作日志里的备注。它会把失败分类、超时责任图、迁移表、人工接管口、执行轨迹当成系统设计的一部分。只要这条线继续写下去，妈妈一个月后的面试素材会非常明显地高出“只会调 API”的候选人一截。

### 2.3 Shadow Mode 与 JD 映射：开始把学习沉淀成求职交付物
`不握钥匙的学徒：Shadow Mode 为什么是 Agent 上线前的成人礼` 和 `岗位 JD 反推表：30分钟补一张 Agent 面试能力映射` 说明另一件很好的事：这周已经不只是学知识，而是在主动把知识压成“可求职交付物”。

前者是上线前的灰度放权逻辑，后者是岗位能力拆解。一个偏系统设计，一个偏求职转译。两篇放在一起，刚好构成“我会做什么”和“我怎么证明自己会做什么”这两个面。

**本周 AI 线结论：** 已经从“学会几个术语”进入“能把权限、恢复、观测、演练、交付物串起来”的阶段。这个方向很对。

## 3. JLPT N2 周复盘：重新开动了，但复习债还压着

这里必须严格按 runbook 说话：**当前阶段没有 AndroidCraft 真实交互反馈，下面的弱项与 boost_words 都只是基于文章与 progress 的推断，不代表妈妈真实掌握度。**

### 3.1 本周到底生成了几篇 N2 每日文章？
按任务要求先检索本周 `_Language/*jlpt-n2-30words*.md`：**0 篇**。

但本周实际运行产物已经切到 10 词模式，真实存在的是：
- `2026-05-14-jlpt-n2-10words.md`
- `2026-05-15-jlpt-n2-10words.md`
- `2026-05-16-jlpt-n2-10words.md`
- `2026-05-17-jlpt-n2-10words.md`

另有 1 篇评估文：
- `2026-05-14-jlpt-n2-learning-evaluation.md`

所以这周“每日词文章”的真实执行量是 **4 篇 10words**，不是 30words。

### 3.2 理论新增词量是多少？实际推进到哪个 index？
如果按本周真正发布的 4 篇 `每日10词` 文章计算：
- **理论焦点词投放量：40 词**

如果按 `focus_history` 看系统侧脚本推进：
- 05/11-05/17 共记录 **7 天**
- 每天 10 个 focus slots，共 **70 个 focus slots**
- 但这 70 个 slots 里有大量重复词、自适应回访词、复习词，不能把它等同于“70 个新词”

目前 `japanese-n2-progress.json` 显示：
- **current_index = 112**
- 词库总量 **390**

也就是说，系统指针现在停在 **112/390**。但这只是脚本进度，不是真实掌握度；runbook 已经明确警告：博客文章和 progress 不能被当作“妈妈已经会了”的证据。

### 3.3 到期复习词有没有积压？
有，而且还不轻：
- **55 个 overdue**
- **10 个今日到期**

这说明 N2 系统的问题已经不在“有没有重新开始”，而在“复习债能不能压下去”。如果 overdue 继续堆，文章照样可以每天发，记忆却不会同步长出来。

### 3.4 哪些标签是明显弱项？
本周 4 篇 `10words` 文章的 machine-readable 小节统计如下：

| weak tag | 出现次数 |
|---|---:|
| abstract | 4 |
| verb | 4 |
| society | 3 |
| communication | 3 |

所以本周最该收敛的弱项标签，就是：
- `verb`
- `abstract`
- `communication`
- `society`

这个结果和 05/14 的学习评估是一致的：妈妈现在更需要“能开口用的动词、能解释关系的沟通词、能读懂抽象表述的基础词”，而不是继续被密集的书面抽象名词轰炸。

### 3.5 哪些具体词应该进入 boost_words？
按本周 4 篇文章的重复建议频次，优先级最高的是：
- `活かす`（4）
- `契約`（4）
- `誤解`（4）
- `条件`（4）
- `主張`（3）
- `制度`（2）
- `情報`（2）
- `水準`（1，但和抽象表达能力直接相关）

这些词已经回写到 `adaptive_profile.boost_words`。原因很简单：它们要么反复出现，要么正好卡在“看得懂一点、自己说不顺”的危险区。

### 3.6 下周应该优先补什么？
下周 N2 线只做三件事：
1. **先清复习债。** overdue 还在 55，就不要谈提速。
2. **保持低负荷 10 词模式。** 当前阶段不适合回到 30 词轰炸。
3. **优先补 verb / communication / abstract。** 让词能被说出来、用出来，而不是只在眼睛里飘过去。

## 6. 一句话总结

这周最值得肯定的，是妈妈在 Android 和 AI 两条线都开始抓“真正决定系统能不能落地”的接口面与边界条件：系统入口、工具合同、错误恢复、灰度放权、迁移表。这些东西一旦沉淀下去，求职素材会越来越硬。

这周最该盯住的风险，是 N2 线还在“能产出文章”和“能形成记忆”之间摇摆。系统已经重新启动，但复习债没有自动消失。下周要做的不是热血加码，而是稳稳把 overdue 压下去。
