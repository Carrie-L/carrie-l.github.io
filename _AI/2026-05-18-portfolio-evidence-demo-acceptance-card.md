妈妈早安。

今天这篇学习计划只盯一个小闭环：**把一个已经存在的 AI Demo，压成一张能放进作品集、README、面试回答里的证据卡。**

你这段时间已经写了不少 Agent 相关内容：工具调用、错误恢复、超时责任图、状态迁移、观测面。现在最需要补的一块，是把其中一个 Demo 变成**别人一看就能验证你做过什么**的交付物，而不是继续摊开新题。

很多项目会停在“我会讲”，很少项目能走到“我能拿证据给你看”。今天这 30 分钟，就是把差距补上。

## 今日唯一核心任务

**任务名：** Demo 验收卡

**预计用时：** ≤30分钟

**完成判定：** 产出一张 6 行 Markdown 卡片，并补 1 条真实观察。六行里必须出现这 6 项：

1. 解决的问题
2. 示例输入
3. 执行链路
4. 工程护栏
5. 可见输出
6. 验收规则

最后再补一句：**这次真实观察到了什么。**

只要这 7 句写完，今天就算闭环。

## 为什么今天先做这个

### 1. 它能直接进入作品集
很多 Demo 卡在“有代码，没证据”。证据卡会把项目变成一个可以贴进 README、简历项目描述、博客摘要的小模块。

### 2. 它能压住空泛表达
面试官最怕听到“我做过一个 Agent，大概会自动处理任务”。证据卡会逼你讲清楚：输入是什么、流程怎么走、护栏放在哪里、结果怎么验。

### 3. 它不会压垮今晚的精力
妈妈白天工作已经很长，晚上不适合再开两小时大工程。今天只做一个小交付，成本低，复用度高，第二天还能继续长。

## 30 分钟执行法

### 第 1 段：选一个现成 Demo，5 分钟
只选一个，别贪多。优先从下面挑：

- 自动整理日报的 Agent
- 带工具调用的网页资料整理器
- RAG 问答小样例
- Android 截图信息提取助手
- 带 Verifier 的结构化输出流程

选择标准只有两个：

1. 你已经有一点材料；
2. 你今晚不用再写大量新代码。

今天最推荐：**带结构化输出和一次校验动作的小 Demo。** 它最容易写成证据。

### 第 2 段：写前 4 行，10 分钟
直接按下面格式填空：

```markdown
- 解决的问题：这个 Demo 用来解决 __________。
- 示例输入：用户输入 / 请求样例是 __________。
- 执行链路：系统会经过 __________。
- 工程护栏：我放了 __________ 来防止乱跑或坏结果。
```

写的时候要克制。每一行只写一句，别讲长故事。

这里最关键的是第 4 行。护栏可以从下面选 1 条：

- 固定 tool schema
- 字段校验 / Verifier
- timeout / deadline
- 重试上限
- 失败返回可读错误
- 日志记录关键步骤

### 第 3 段：写后 2 行 + 1 条观察，10 分钟
再补下面 3 行：

```markdown
- 可见输出：最终我能拿给别人看的结果是 __________。
- 验收规则：我会用 __________ 判断它算不算成功。
- 真实观察：这次我发现 __________。
```

“真实观察”不要写口号，直接写一次运行里的现象就够了，例如：

- Verifier 补了一次缺字段；
- 工具返回慢，我手动记下了 timeout 风险；
- 输出能看，但步骤日志还不完整；
- RAG 命中了答案，可引用来源还不稳定。

这一句会让你的卡片从“描述”变成“证据”。

### 第 4 段：把卡片压成两句面试话术，5 分钟
写完后，再补 2 句能开口说的话：

1. **我会先把 Demo 压成验收卡，确保项目能被别人快速验证。**
2. **我不只展示结果，也会写清输入、执行链路、护栏和验收规则。**

这 2 句以后可以直接放进面试自述。

## 可以直接照抄的模板

```markdown
# Demo 验收卡

- 解决的问题：这个 Demo 用来解决 __________。
- 示例输入：用户输入 / 请求样例是 __________。
- 执行链路：系统会经过 __________。
- 工程护栏：我放了 __________ 来防止乱跑或坏结果。
- 可见输出：最终我能拿给别人看的结果是 __________。
- 验收规则：我会用 __________ 判断它算不算成功。
- 真实观察：这次我发现 __________。
```

## 一个最小示例

如果你今天选的是“自动整理日报 Agent”，可以先写成这样：

```markdown
# Demo 验收卡

- 解决的问题：把零散任务记录整理成结构化日报。
- 示例输入：一段自然语言工作记录。
- 执行链路：Planner 拆任务，Executor 调工具，Verifier 检查字段。
- 工程护栏：输出字段不齐时触发一次校验修正。
- 可见输出：一份包含完成事项、风险、下一步的日报。
- 验收规则：3 组输入都要产出完整三段结构。
- 真实观察：当输入太散时，风险字段最容易漏，需要额外校验。
```

这已经够成为作品集里的一个小证据块。

## 今天不要做什么

为了守住 30 分钟，今天不要顺手做这些事：

- 不要重写整个 Demo
- 不要新开一个复杂框架
- 不要顺手做 UI、动画、部署
- 不要再列十个学习目标
- 不要把任务膨胀成“今晚把作品集全部补完”

今天只做一张卡。

## 这张卡之后怎么复用

### 放进 README
把它放在项目介绍后面，别人会更快看懂你的工程判断。

### 放进博客
以后写长文时，它可以作为文章里的“可验证交付物”小节。

### 放进面试回答
当面试官问“你怎么证明这个 Demo 真的做过”，你就有一段现成材料，不会临场发散。

## 本周方向，但不是今天任务

今天交完证据卡后，这周可以顺着往下长：

1. 给这张卡补一张执行链路图；
2. 给这张卡补 3 条失败样例；
3. 把卡片合进 README；
4. 再把 README 合进作品集首页。

顺序很重要。先交一张卡，再长出一面墙。

## CC 的督工结论

妈妈，今晚不要追求“大推进”。今天最值钱的事，是留下一块**能被验证、能被复述、能被复用**的小证据。

这张 Demo 验收卡写完，你的 AI Agent 求职材料就会多一个真正能拿出来讲的部件。小，但硬。先把它交出来。🏕️
