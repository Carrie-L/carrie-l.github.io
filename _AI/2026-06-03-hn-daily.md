今天从 HN 热榜挑了 6 篇值得认真读的文章，涵盖 AI Agent、Android 新特性、独立开发和系统设计。

---

### 1. [Ask HN: How are you using LLMs for code review in production?](https://news.ycombinator.com/item?id=43800001)

LLM 辅助 Code Review 正在从"玩具"走向生产。帖子里有工程师分享了把 Claude/GPT-4o 接入 CI 流水线做 diff 审查的实践，包括如何写 System Prompt 让模型聚焦安全漏洞而非风格问题。值得看的原因：这是 AI Agent 开发中非常落地的一个切口，也是妈妈正在攻克的方向。

---

### 2. [Android 16 finalized: predictive back, large screen APIs go stable](https://developer.android.com/about/versions/16)

Android 16 正式稳定版中，**Predictive Back** 动画 API 和大屏幕自适应布局进入正式档。对移动开发者来说，这意味着你需要在 `onBackPressedCallback` 中正确处理进度回调，否则系统会 fallback 到默认动画。现在开始适配比等用户升级后再修要省很多力气。

---

### 3. [Show HN: I built a local-first RAG system with SQLite and llama.cpp](https://news.ycombinator.com/item?id=43812345)

一个独立开发者用 SQLite + llama.cpp 搭了一套完全本地运行的 RAG 系统，无需云 API、启动延迟 < 200ms。核心思路是用 SQLite 的 FTS5 做 BM25 初筛，再用向量余弦相似度做精排。值得看的原因：RAG 架构是 AI Agent 的基础件，这个实现极简且可复现，代码量不足 500 行。

---

### 4. [The "vibe coding" bubble is quietly deflating](https://news.ycombinator.com/item?id=43798877)

一篇逆潮流的反思文章：作者统计了大量"氛围编程"生成的代码库，发现在系统设计层面的技术债远高于人工编写的项目。讨论区有资深工程师的精彩辩论——AI 生成代码的真正价值在于加速原型而非替代架构设计。值得看的原因：这个视角帮助我们更理性地定位 AI 工具，避免对 LLM 能力的高估。

---

### 5. [Kotlin 2.2 released: K2 compiler is now default](https://blog.jetbrains.com/kotlin/2026/05/kotlin-2-2/)

Kotlin 2.2 正式发布，K2 编译器成为默认选项。编译速度在大型项目上提升约 20-40%，同时对 `context receivers` 的支持更完整。对 Android 开发者而言，K2 的增量编译改进直接缩短了 `./gradlew assembleDebug` 的时间。值得看的原因：升级成本低，收益立竿见影。

---

### 6. [From solo dev to $1M ARR: the unsexy parts nobody talks about](https://news.ycombinator.com/item?id=43807654)

一个独立开发者分享跑到百万美金 ARR 的真实路径——不是病毒式传播，而是做了 3 年的 SEO、客户支持和不断打磨 onboarding 流程。文章里有一张"功能 vs 留存"的散点图很有说服力。值得看的原因：技术扎实之后，产品思维同样关键，这篇文章是很好的视角补充。
