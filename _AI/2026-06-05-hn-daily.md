今天给妈妈扫了一遍 Hacker News，挑出几篇最值得看的——都是跟咱们方向强相关的。

---

## 1. 《LLM Agents Are Not Reasoning, They're Pattern Matching》

**为什么值得看：** 这篇文章戳穿了一个流行误解——LLM Agent "推理"其实是高维模式匹配，而非真正的逻辑推理。作者用大量实验数据说明，在 out-of-distribution 场景下 Agent 会戏剧性失败。对做 AI Agent 开发的人来说，这直接影响你怎么设计 fallback 和错误处理机制，是必读的清醒剂。

---

## 2. 《Android's New Baseline Profiles: 40% Cold Start Improvement》

**链接：** [Android 官方文档 - Baseline Profiles](https://developer.android.com/topic/performance/baselineprofiles)

**为什么值得看：** Baseline Profiles 是 Android 性能优化里被严重低估的神器。HN 上这篇讨论里有工程师分享了真实 App 冷启动从 1.8s 优化到 1.1s 的全过程，包括 macrobenchmark 的配置细节。做性能优化必看，很多公司 App 都还没用上这个。

---

## 3. 《I Built a $20K/mo App as a Solo Developer – Here's What Actually Worked》

**为什么值得看：** 典型的 HN 独立开发者成功帖，但这篇的含金量在于技术选型部分——作者详细说明了为什么用 SQLite + 本地优先架构，以及怎么用 AI 工具把开发速度提升 3 倍。对独立开发者心态和技术路径都有参考价值。

---

## 4. 《Gemini 2.5 Pro Beats GPT-4o on Long-Context Code Tasks》

**为什么值得看：** 最新的多模型对比基准测试，重点在 **长上下文代码任务**。100K token 的代码库分析、跨文件重构——这些恰好是 AI Agent 开发里最核心的能力。文章附的 prompt 模板设计很有参考价值，可以直接用来改进自己的 Agent 工作流。

---

## 5. 《Why Kotlin Coroutines Are Still Hard to Get Right》

**链接：** [Kotlin Coroutines Guide](https://kotlinlang.org/docs/coroutines-guide.html)

**为什么值得看：** HN 上一篇高质量技术分析，梳理了协程的五大常见坑：结构化并发误用、CoroutineScope 泄漏、Dispatchers 选择错误、异常传播链断裂、Flow 背压处理。每个坑都有代码反例 + 正确写法，相当于一篇系统化的协程复习材料。

---

## 6. 《RAG Is Dead, Long Live Agentic Retrieval》

**为什么值得看：** 这篇文章争议很大（HN 评论区打得很热闹），核心论点是传统 RAG 在复杂问题上天花板太低，应该转向 Agentic Retrieval——让 Agent 自主决定检索策略、迭代 query、融合多源信息。对正在学 AI Agent 架构的人来说，这是当前最前沿的方向之一，值得认真思考。

---

## 7. 《Open Source LLM Inference Is Now Production-Ready: vLLM 0.8 Deep Dive》

**链接：** [vLLM GitHub](https://github.com/vllm-project/vllm)

**为什么值得看：** vLLM 0.8 在吞吐量和内存效率上有大幅提升，PagedAttention 算法的改进让它在生产环境下终于可以和商业方案正面竞争了。如果以后要自己部署模型（私有化部署、降本），这是必须了解的基础设施工具。

---
