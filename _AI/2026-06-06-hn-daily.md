## 1. LangGraph：构建有状态 AI Agent 的核心框架

**链接**：[github.com/langchain-ai/langgraph](https://github.com/langchain-ai/langgraph)

LangGraph 把 Agent 工作流建模成一个有向图——节点是"动作"，边是"条件路由"，整个对话状态随着图的遍历不断演进。相比 ReAct 模式的线性循环，它能优雅地表达多轮反思、并行子任务、回退重试等复杂流程。对于想深入 Agent 架构的工程师来说，LangGraph 的源码本身就是一篇关于状态机设计的好教材。

---

## 2. Android Baseline Profiles：启动时间压缩 30%

**链接**：[developer.android.com/topic/performance/baselineprofiles](https://developer.android.com/topic/performance/baselineprofiles)

Baseline Profiles 允许开发者把关键代码路径的 AOT 编译规则打包进 APK，ART 在首次安装时直接按规则预编译，跳过解释执行阶段。Google 的测量数据显示，合理配置后冷启动和首帧渲染时间可缩短 20-30%。文章附带了 Macrobenchmark 接入教程，是近期 Android 性能优化必读之一。

---

## 3. Qwen2.5-Coder：开源代码大模型的新标杆

**链接**：[github.com/QwenLM/Qwen2.5-Coder](https://github.com/QwenLM/Qwen2.5-Coder)

这款专为代码任务训练的开源模型在 HumanEval、MBPP 等基准上表现亮眼，32B 参数版本在多项指标上接近闭源 GPT-4o-mini。对独立开发者来说更实际的意义是：可以本地部署，不用担心代码隐私问题，配合 Continue 或 Aider 等工具就能搭出私有代码助手。

---

## 4. 用 Rust 重写工具链——独立开发者的性能哲学

**链接**：[github.com/ast-grep/ast-grep](https://github.com/ast-grep/ast-grep)

ast-grep 是一个用 Rust 编写的结构化代码搜索/重构工具，用 AST 模式匹配代替正则表达式。一行命令就能在整个项目里找出所有"特定形状"的代码块并批量改写。作者以个人开发者身份维护，在 HN 引发了"Rust 工具链是否值得所有基础软件重写"的经典讨论。对想入门 Rust 或研究 AST 的工程师有参考价值。

---

## 5. Gemma 3n：Google 为边缘设备设计的多模态小模型

**链接**：[ai.google.dev/gemma](https://ai.google.dev/gemma)

Gemma 3n 采用了 MatMul-Free 等激进压缩手段，在手机 NPU 上实现实时推理，支持文本+图像输入。对 Android 工程师而言意义重大——这标志着设备端大模型进入实用阶段，未来 App 的 AI 功能可以完全离线运行，不再依赖云 API。

---

## 6. Show HN：用 SQLite 实现简单向量搜索

**链接**：[github.com/asg017/sqlite-vec](https://github.com/asg017/sqlite-vec)

sqlite-vec 是一个 SQLite 扩展，把向量相似度检索塞进了你最熟悉的 SQL 查询里。对于不想引入 Pinecone/Weaviate 等重型向量数据库的独立开发者，这是搭建轻量 RAG 系统的最低门槛选项。作者在 HN 的帖子里详细解释了实现原理，评论区有关于"向量数据库是否过度工程化"的精彩辩论。

---

## 7. Papers With Code：Transformer 注意力机制可视化合集

**链接**：[paperswithcode.com](https://paperswithcode.com)

近期 paperswithcode 上有几篇关于注意力头功能分解的新论文，用可视化方式展示了不同注意力头分别在做什么——有的负责语法依存，有的负责实体共指。对想从原理层面理解 LLM 为什么"读得懂"文字的工程师来说，这类机制可解释性研究既烧脑又迷人。
