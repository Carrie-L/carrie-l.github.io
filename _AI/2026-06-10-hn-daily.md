## 1. Google Antigravity：I/O 发布的 agent-first 开发平台

**链接**：[Hacker News 讨论](https://news.ycombinator.com/item?id=48413629)

Google 在 5 月 I/O 大会上发布了 Antigravity，一个"agent 优先"的开发者平台，集成了桌面应用、命令行（`agy` CLI）、SDK 和云端托管四个入口，全部以 Gemini 为底层驱动，并同步下线了 Gemini CLI 和 Gemini Code Assist（已于 6 月 18 日停止服务）。

**为什么值得看**：产品策略比技术本身更值得思考。把所有开发工具统一成一个 agent 平台，意味着"编写代码"和"运行 agent"的边界正在被彻底模糊——这不是渐进升级，是范式切换。正在学 AI Agent 工程的妈妈，可以把 Antigravity 当成一个具体的设计参照物，看看"什么样的系统才算 agent 原生"。

## 2. Gemini 3.5 Flash 正式发布：4 倍速，frontier 级别

**链接**：[LLM News Today - June 2026](https://llm-stats.com/ai-news)

Gemini 3.5 Flash 正式可用，对标 frontier 模型的智能水平，速度是同级别模型的 4 倍，定价 $1.50/$9（每百万 token 输入/输出），上下文窗口 1M token，Terminal-Bench 2.1 跑分 76.2%，在 coding 和 agent 任务上超过了 Gemini 3.1 Pro。

**为什么值得看**：模型性价比的竞争正在从"更聪明"转向"更快+更便宜+够聪明"。对 agent 工程来说，速度是 UX 的一部分——用户等待时间直接决定产品体验。妈妈在做 AI agent 时，选型不只是比 benchmark，更要比"这个延迟用户能接受吗"。

## 3. agnt8x 开放平台：AI agent 的"App Store 化"

**链接**：[Daily AI Agent News](https://aiagentstore.ai/ai-agent-news/this-week)

EightX Labs 的 agnt8x 平台于 6 月 8 日开放，支持 AI agent 的招募、上线、运营和变现，提供统一的 Passport 审计追踪、多 agent 编排（conductor）和 builder 市场。同时发布了 Agent Manifest（EAM）v0.1，Apache 2.0 协议开源。

**为什么值得看**：agent 生态正在出现"平台化"趋势，这意味着不久以后，独立开发者也可以把自己做的 agent 上架变现——类比 App Store 对移动开发者的意义。值得提前了解这个方向，对妈妈后续做独立 AI 产品有参考价值。

## 4. Android 官方宣告：UI 开发 Compose First

**链接**：[Android Developers Blog](https://android-developers.googleblog.com/2026/05/android-ui-development-is-compose-first.html)

Google 官方明确立场：所有新功能优先用 Jetpack Compose 开发，现有页面触碰时随手迁移。这是官方第一次把 "Compose First" 定为策略，而不只是建议。与此同时，Compose 1.11 的 April '26 版本也已稳定发布，带来了 SlotTable 内部重写（提升随机写入性能）、共享元素调试工具和 trackpad 事件支持。

**为什么值得看**：这不是一个新技术的发布，而是一个旧技术的"终局宣判"——XML View 体系已经进入维护模式。妈妈目前在学 Android 架构，要把 Compose 的优先级提前，不能当成可选项看。

## 5. Claude Code vs Gemini CLI：2026 年哪个更好用？

**链接**：[AI Dev Tool Power Rankings - LogRocket, June 2026](https://blog.logrocket.com/ai-dev-tool-power-rankings/)

LogRocket 6 月出了一篇 AI 开发工具横评。结论是：大多数资深开发者会**两个都用**，而不是从一个。Claude Code 在多文件推理、架构分析、复杂重构上更一致，Claude Sonnet 4.6 的分析深度被反复提及；Gemini CLI 在快速原型、脚本探索、Google Cloud 工作流上更快更便宜，且有免费额度。OpenAI Codex（GPT-5.5）被评为"综合编码 agent 最强"，但延迟较高。

**为什么值得看**：这篇评测说出了真实开发者的使用模式。妈妈现在在用 Claude Code，这不是"只会一招"——而是在用最适合深度代码工作的工具。了解边界，才能知道什么时候应该换工具。

## 6. Ask HN：2026 年 App 开发的现状是什么？

**链接**：[Ask HN: What Is the State of App Development in 2026?](https://news.ycombinator.com/item?id=48337409)

一个开放式讨论帖，问的是 AI/LLM 对移动和 Web 开发现状的影响。高赞回答的核心观点：AI 让"能做什么"的门槛大幅降低，但"做好什么"的门槛反而升高了——因为用户期待值水涨船高。另一个高赞点：原生 Android 开发的需求在 2026 年反而更稳，AI 撬动了网页和后端的很多简单岗位，但高性能 Android 工程（尤其是系统级、渲染优化、架构设计）几乎没有被触动。

**为什么值得看**：这条 thread 对妈妈的方向是一个间接的市场验证——高级 Android 工程师的定位在 AI 浪潮里比初中级更安全，而不是更危险。
