## 1. LLM 上下文工程正在取代 Prompt 工程

**关键词：Context Engineering / Agent**

"Prompt engineering is dead, long live context engineering" 这类讨论在 HN 上持续高热。核心观点是：现代 LLM 的能力瓶颈不在于你怎么"说话"，而在于你给它多少**结构化上下文**——工具定义、记忆片段、当前状态、历史轨迹。RAG、Tool Calling、Memory 模块的组合方式，决定了 Agent 系统的天花板。

👉 值得关注，因为这直接关系到 AI Agent 工程实践。

---

## 2. Kotlin Multiplatform 1.x 正式稳定

**关键词：KMP / Android / iOS**

KMP 从实验走向稳定后，HN 的 Android 开发者社区开始认真讨论"共享业务逻辑"的边界在哪里。UI 层仍然分离（Compose on Android，SwiftUI on iOS），但 Repository、ViewModel、网络层共享已经是实践共识。JetBrains 的推进节奏超出预期。

👉 Android 进阶必须了解的跨平台架构走向。

## 3. 独立开发者的 AI 工具栈 2026

**关键词：Indie Hacker / AI-assisted Dev**

HN 上涌现了一批"一个人 + AI 工具 = 完整产品"的故事。典型栈：Claude Code 做代码生成、Cursor 做日常编辑、Supabase 做后端、Vercel 做部署。一位开发者 3 周内上线了一个订阅制 SaaS，首月 MRR 破千美元，帖子在 HN 上获得大量讨论。

👉 对独立开发者和副业探索者有很强的参考价值。

## 4. Android 内存分析工具的演进：MAT vs Android Studio Profiler

**关键词：Android / Memory / OOM**

一篇技术贴详细比较了 Eclipse MAT 和 Android Studio Memory Profiler 在实际 OOM 排查中的差异。结论是：Studio Profiler 的 Heap Dump 分析已经足够日常使用，但 MAT 在处理超大 heap（>500MB）时仍然是王者。文中还提到了 LeakCanary 3.x 的新特性。

👉 Android 内存优化进阶必备知识点。

---

## 5. "Vibe Coding" 的反思：AI 生成代码的技术债务

**关键词：AI 编码 / 技术债务 / 代码质量**

这是 HN 上一篇争议性很强的帖子。作者认为，AI 生成的代码表面上跑通了，但缺乏对业务不变量的理解，导致大量隐性技术债。评论区分成两派：一派认为工程师需要提升审查能力；另一派认为这是 AI 工具本身的局限。

👉 用 AI 工具开发的工程师必读，培养批判性代码审查意识。

---

## 6. Android Compose 性能优化：Stability 与重组控制

**关键词：Jetpack Compose / Recomposition / 性能**

HN 链接到了一篇深度分析 Compose Stability 机制的文章。核心结论：未被标记为 `@Stable` 或 `@Immutable` 的数据类会导致不必要的 recomposition，严重影响列表性能。通过 Compose Compiler Report 可以量化分析组件的稳定性。

👉 写 Compose 必须懂的底层机制，妈妈正在学这个！

---

## 7. SQLite 作为服务端数据库的崛起

**关键词：SQLite / Serverless / Turso**

2026 年越来越多的小型 SaaS 开始用 SQLite（通过 Turso/LiteFS）作为生产数据库。HN 讨论的核心是：嵌入式数据库 + 边缘部署的组合，对独立开发者来说成本极低，延迟极小。这是一个值得关注的技术趋势转变。

👉 架构选型视野的拓展，了解不同场景下的数据库权衡。
