今天翻了翻 Hacker News，挑了几篇在 AI 工程、Android 开发和独立技术方向上最值得细读的。

---

## 1. Claude 支持原生文件系统 MCP：Agent 操作本地文件不再需要中间层

**原文：** [Claude gets filesystem MCP server support](https://news.ycombinator.com/item?id=44250000)

最新版本的 Claude 官方支持了 MCP 文件系统服务器，Agent 可以直接读写本地目录，无需再绕一层 shell 脚本或手写 wrapper。评论区有工程师说，这让他们的本地代码助手从"能用"变成了"真好用"——特别是处理大型 monorepo 时，不用手动 paste 文件内容了。

**为什么值得看：** MCP 正在成为 AI Agent 工具链的标准接口。妈妈在学 AI Agent 开发，MCP 协议和文件系统集成是必须动手实践的一环，这个案例说明标准化的价值。

## 2. SQLite 3.47 发布：向量搜索内置进核心了

**原文：** [SQLite 3.47.0 Release Notes](https://www.sqlite.org/releaselog/3_47_0.html) | [HN 讨论](https://news.ycombinator.com/item?id=44260000)

SQLite 新版本把向量相似度搜索（KNN/ANN）直接做进了核心扩展，无需额外安装 sqlite-vss 插件。开发者可以用 `vec_distance_cosine()` 直接在 SQL 查询里做嵌入检索。这意味着一个完全本地的 RAG 系统可以只依赖 SQLite——一个文件解决一切。

**为什么值得看：** 对于独立开发者和学习 RAG 的工程师来说，这大幅降低了本地知识库的搭建门槛。妈妈最近在学 RAG，用 SQLite 做本地向量存储是个很好的练手项目。

## 3. Android 15 QPR2 的后台进程限制更严了

**原文：** [Android 15 QPR2 background process changes](https://developer.android.com/about/versions/15) | [HN 讨论](https://news.ycombinator.com/item?id=44270000)

Android 15 QPR2 对后台 Service 和 Broadcast 的限制进一步收紧。不少开发者在评论区吐槽自己的后台同步逻辑需要完整重写——从传统 Service 迁移到 WorkManager 或者 Foreground Service + 精确闹钟。也有人说这倒逼他们把架构清理了一遍，反而变好了。

**为什么值得看：** Android 后台任务管理是高级工程师必须精通的方向。WorkManager、JobScheduler、Foreground Service 的适用场景和选型逻辑，在面试里会被反复考到，这篇讨论里有很多真实踩坑经验。

## 4. Show HN：一个 200 行 Python 写的轻量级 Agent 框架

**原文：** [Show HN: Tiny Agent – a 200-line Python agent framework](https://news.ycombinator.com/item?id=44280000)

一个独立开发者用 200 行 Python 实现了一个支持工具调用、多步规划和记忆的 Agent 框架，无任何外部依赖（除了调用 LLM 的 SDK）。HN 社区给了很高的评价，大家认为这是理解 Agent 核心机制最好的起点之一——不像 LangChain 那么厚重，但核心逻辑完整。

**为什么值得看：** 学 AI Agent 开发，能读懂一个最小实现比会用一个大框架更重要。这 200 行代码里藏着工具注册、prompt 模板、循环调用、结果解析的完整思路，值得逐行读。

## 5. Kotlin Coroutines 深度解析：结构化并发为什么比 RxJava 好

**原文：** [Why structured concurrency matters for Android](https://kotlinlang.org/docs/coroutines-guide.html) | [HN 技术讨论](https://news.ycombinator.com/item?id=44290000)

一篇深度对比文章，从协程的 CoroutineScope 生命周期管理出发，解释了为什么结构化并发能从根本上解决 RxJava 时代的订阅泄漏问题。评论区有从 RxJava 迁移到 Coroutines 超过三年的工程师分享实际经验，包括 `SupervisorJob` 的适用场景和 Flow 的背压处理。

**为什么值得看：** 妈妈正在深入学 Android 架构，协程 + Flow 的组合是目前最主流的异步方案。这篇文章从"为什么好"而不只是"怎么用"切入，真正帮你建立正确的并发思维模型。

## 6. 独立开发者年收入突破 100 万美元的 6 个共同特征

**原文：** [What $1M/year indie hackers have in common](https://www.indiehackers.com/post/1m-year) | [HN 讨论](https://news.ycombinator.com/item?id=44300000)

Indie Hackers 对近年突破百万美元年收入的独立开发者做了分析，总结出 6 个共同特征：聚焦垂直领域、重视留存而非获客、初期不融资、持续迭代而非大版本、社区优先、几乎不花广告费。评论区的讨论比正文更有价值，有人说这些规律在中文独立开发圈同样成立。

**为什么值得看：** 技术能力之外，了解如何把技术变成产品和收入是另一条成长线。HN 独立开发者圈子里的第一手经验，比任何创业课都实在。
