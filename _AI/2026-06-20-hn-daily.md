今天的 HN 首页充满了 AI Agent 基础设施的讨论——从状态机到本地模型选型，从安全漏洞到开发者工作流，整个社区都在认真思考"AI 究竟怎么用好"这件事。

---

## 1. Show HN: Git for AI Agents

🔗 [news.ycombinator.com/item?id=48063548](https://news.ycombinator.com/item?id=48063548)

有人把 Git 的版本管理思想移植到了 AI Agent 的执行过程里——每次 Agent 做决策，都像一次 commit，可以回滚、可以 diff、可以 branch 到不同执行路径。这是个非常有趣的方向：Agent 不再是一条黑箱流水线，而是可审计、可调试的流程图。对于想严肃构建 AI 产品的工程师，这个思路值得深挖。

---

## 2. Show HN: Statewright – 用可视化状态机让 AI Agent 更可靠

🔗 [news.ycombinator.com/item?id=48108778](https://news.ycombinator.com/item?id=48108778)

Statewright 把 AI Agent 的行为建模为显式状态机——每个状态、每条转移边都是可视的、可编辑的。这直接戳中了当前 Agent 框架的最大痛点：LLM 的输出概率性太强，纯靠 prompt 控制流程太脆。把确定性逻辑和 LLM 的生成能力分开处理，是一种很成熟的工程思维。如果你在搭 Agent，这个工具值得试。

---

## 3. Show HN: 根据你的硬件找最适合的本地 LLM

🔗 [news.ycombinator.com/item?id=48146369](https://news.ycombinator.com/item?id=48146369)

一个工具，输入你的显存/内存配置，输出最适合跑的本地模型排行榜。在本地 LLM 越来越可用的今天，"什么模型配什么硬件"成了真实的工程问题。这个工具背后是真实 benchmark 数据，不是拍脑袋推荐——对想在移动端或嵌入式场景跑模型的开发者也有参考价值。

---

## 4. Ask HN: 你的 AI 开发技术栈和工作流是什么？

🔗 [news.ycombinator.com/item?id=48413629](https://news.ycombinator.com/item?id=48413629)

一个高质量的讨论帖。2026 年的主流 AI 开发者画像正在浮现：Python + TypeScript 双栖、向量数据库 + RAG 是标配、Claude/GPT/Gemini API 会混用、MCP 协议开始成为工具集成的标准通道。回复里有很多真实的生产经验，比看教程有用得多。

---

## 5. Ask HN: 2026 年 App 开发现在是什么状态？

🔗 [news.ycombinator.com/item?id=48337409](https://news.ycombinator.com/item?id=48337409)

这个帖子问的是移动开发的整体生态——原生 vs 跨平台、AI 辅助开发的边界在哪里、独立开发者还有没有机会。讨论很务实，没有宣传腔。结论大致是：原生开发（Android/iOS）在性能敏感场景依然不可替代，但 AI 工具确实把个人开发者的效率提升了一个数量级。

---

## 6. Ask HN: 你用 GenAI 时的"卧槽"时刻是什么？

🔗 [news.ycombinator.com/item?id=48406174](https://news.ycombinator.com/item?id=48406174)

有趣的感性讨论。很多工程师分享了第一次被 LLM 惊到的场景——不是"它答对了答案"，而是"它理解了我没说清楚的意图"。这种体验背后是 in-context learning 和 attention 机制的力量。读这个帖子会帮你重新理解为什么 LLM 不只是搜索引擎。

---

## 7. AutoJack：AI Agent 的 MCP 安全漏洞

🔗 [news.ycombinator.com/](https://news.ycombinator.com/)

研究人员披露了一个名为 AutoJack 的漏洞利用链，能把 AI 浏览 Agent 变成远程代码执行的载体——漏洞位于一个流行的多 Agent 框架的 MCP 路由层。随着 MCP 协议普及，AI 工具的攻击面正在快速扩大。如果你在开发或部署 AI Agent，安全模型和最小权限原则现在就要纳入设计。
