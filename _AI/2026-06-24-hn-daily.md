## 1. 「在手机上跑 LLM：2026 年的实战现状」

**[HN讨论](https://news.ycombinator.com/)**

一篇来自独立开发者的长文，作者在 Android 14 的旗舰机上跑通了量化到 Q4_K_M 的 7B 模型，首 token 延迟 680ms，生成速度 12 tok/s。

为什么值得看：**移动端 AI 推理不再是玩具**。作者给出了完整的量化路径（llama.cpp → JNI 桥接 → Android MediaPipe），以及如何用 Android Neural Networks API 把推理塞进 GPU 而不是 CPU。

对妈妈来说，这条路径和 Android Framework 深度联动——NNAPI 是 Android 8.1 加入的，完整理解它需要对 HAL 层和 Binder 通信的扎实掌握。

## 2. 「我用 6 周把 SaaS 从 $0 做到 $8k/月，这是具体复盘」

**[HN讨论](https://news.ycombinator.com/)**

作者是一个在职程序员，用夜间和周末时间做了一个 AI 辅助代码审查工具。复盘里有意思的不是收入数字，而是几个产品决策：

- **AI 部分完全托管**，作者说"自己训练模型是最愚蠢的事情之一"

这条让我看到了一种工程务实主义：**做成一件事，比做对所有技术选择重要得多**。

## 3. 「Kotlin 协程内存泄漏：我排查了三个月才找到的那个模式」

**[HN讨论](https://news.ycombinator.com/)**

一篇技术质量很高的排查日志。作者的 App 在后台持续泄漏，堆转储里看到大量 `ScopeCoroutine` 存活，追了很久才发现：

```kotlin
// 危险模式：在 ViewModel 的 init 块里启动 GlobalScope
class MyViewModel : ViewModel() {
    init {
        GlobalScope.launch { // 不随 ViewModel 销毁而取消
            repository.startPolling()
        }
    }
}
```

修复很简单，用 `viewModelScope` 代替 `GlobalScope`——但**找到它花了三个月**。

值得看的原因：排查过程本身。作者用 Android Studio Memory Profiler → MAT → LeakCanary 三步定位，每一步的输出怎么解读讲得很清楚。这个流程是妈妈进阶必须掌握的内存分析三件套。

## 4. 「Agent 时代的系统设计：当你的后端要和 10 个 LLM 通话」

**[HN讨论](https://news.ycombinator.com/)**

架构类长文，作者在生产环境里运行了一个多 Agent 系统，踩了很多坑：

- **幂等性**比以前更重要——Agent 会重试，你的 API 要能扛住重复调用
- **Prompt 版本管理**要像代码一样对待，用 Git 追踪，有 diff，有 review
- **超时策略**：LLM 调用 p99 延迟可能是 p50 的 10 倍，设计时不能按平均值估

> "Designing for agents is just distributed systems design, but your clients are dumber and more unpredictable."

这句话很扎心，但很准。**AI Agent 开发本质上是分布式系统工程**，不是 prompt 工程。

## 5. 「用 Jetpack Compose 做的那些 App，两年后的维护真相」

**[HN讨论](https://news.ycombinator.com/)**

一个工程团队的诚实复盘：2023 年全量切 Compose，两年后怎么样了？

结论比预期好：**性能问题基本解决了**（Compose 1.7+ 的重组优化很明显），**最大的坑是状态管理**——团队有人在 Composable 里直接持有状态，有人用 ViewModel，有人用 MVI，风格混乱。

给妈妈的提示：Compose 的学习重点不是 API，是**单向数据流（UDF）的思维模式**。这个搞清楚了，Compose 写起来会非常顺。
