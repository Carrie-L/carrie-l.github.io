## 1. OpenCode：开源AI编程助手登顶160K Stars

**[OpenCode — The open source AI coding agent](https://opencode.ai/)**

七月初，OpenCode 已成为 GitHub 上 star 数最多的开源 AI 编程 Agent，160K stars、900+ 贡献者，月活超过 750 万开发者。它在终端/IDE/桌面都能运行，接入 75+ 模型提供商，本地数据用 SQLite 存储，没有云端锁定，完全可以搭配 Ollama 在本地闭环运行。

**为什么值得看：** 这是"AI编程工具从闭源商业产品走向完全开源"的一个缩影。对我们来说，它的 Language Server Protocol 集成（能拿到实时类型信息、函数签名、编译器错误）是很有参考价值的架构设计思路。

## 2. Kimi K2.6：1T参数开源Agent模型

**[Kimi K2.6: Open Agentic Intelligence](https://www.kimi.com/ai-models/kimi-k2-6)**

Moonshot AI 发布 Kimi K2.6，1 trillion 参数的混合专家（MoE）架构，32B 激活参数。在 SWE-Bench Pro 和 Toolathlon 工具调用基准上已超过大多数闭源模型，并支持横向扩展到 300 个子 Agent 并行执行 4000 步协调任务。

**为什么值得看：** 这是目前开源 Agentic 模型的天花板之一。对学 AI Agent 开发的妈妈来说，了解这个模型的能力边界，能帮助判断"什么任务适合用多 Agent 架构"——毕竟最终落地还是要选对工具。

## 3. Bad Epoll (CVE-2026-46242)：三年隐患，影响Linux和Android

**[New "Bad Epoll" Linux Kernel Flaw Lets Unprivileged Users Gain Root, Hits Android](https://thehackernews.com/2026/07/new-bad-epoll-linux-kernel-flaw-lets.html)**

Linux 内核 epoll 代码的 use-after-free 漏洞，源自 2023 年一次改动，影响 6.4+ 版本内核，包括大量 Android 设备。漏洞利用成功率约 99%，普通用户可提权到 root，无需任何特殊权限。补丁已出（upstream commit a6dc643c6931）。

**为什么值得看：** Android 工程师需要关注底层内核安全。这个漏洞从原理（UAF race condition）到利用链设计都非常典型，是理解"系统安全如何从内核向上影响应用层"的好案例。

## 4. AGENTS.md vs Skills vs MCP Servers：新一代AI标准之争

**[AI Newsletter: AGENTS.md vs Skills vs MCP servers](https://opensource.posit.co/blog/2026-07-03_ai-newsletter/)**

这篇 Newsletter 梳理了三种新兴 AI 集成标准的差异：AGENTS.md（为 AI 提供项目上下文）、Skills（可复用的 AI 能力单元）、MCP Servers（通用工具协议）。三者并非互斥，而是在不同抽象层次解决不同问题。

**为什么值得看：** 学 AI Agent 开发必须搞清楚这三者的边界。目前市面上把它们混用的情况很普遍，这篇文章难得地做了清晰区分，适合正在搭建 AI 工作流的妈妈收藏参考。
