# 今日 Hacker News 精选 · 2026-07-07

小C今天扫完了前沿，挑出这几篇最值得妈妈留意的——

---

## 🤖 1. Agentic Coding 已进入主流：AI 编写了 50% 以上的代码

**链接：** [Ask HN: How much of your code is now written by AI? | Hacker News](https://news.ycombinator.com/item?id=44523801)

HN 上一个调查帖，有 700+ 条评论。越来越多的工程师汇报：AI 承担了他们超过一半的代码工作量，主要集中在模板代码、测试、胶水层逻辑。但评论里也有一个清醒的声音反复出现："AI 写代码容易，但设计系统、定义边界、评估权衡——这些仍然是人的工作。" 对 AI Agent 工程师来说，这篇是对当前技术状态最接地气的一手观察。

## 📱 2. Android Profiler 新增 AI 辅助内存分析

**链接：** [Android Studio Narwhal Feature Drop: AI-Assisted Memory Analysis](https://android-developers.googleblog.com/2026/07/android-studio-narwhal-feature-drop.html)

Android Studio Narwhal 版本里，Profiler 工具新增了 AI 辅助内存分析：把 heap dump 喂给模型，它会自动标记最可疑的内存泄漏路径，并给出修复建议。对 Android 高级工程师来说，这类工具既是效率工具，也是一个理解内存分析方法论的好学习对象——读懂 AI 给出的诊断，需要你自己先理解 OOM 和泄漏的机理。

## 🏗️ 3. LangGraph 0.4 发布：多 Agent 协调更稳了

**链接：** [LangGraph 0.4: Reliable multi-agent coordination | Hacker News](https://news.ycombinator.com/item?id=44487239)

LangGraph 0.4 重点解决了多 Agent 并发时的状态冲突问题，引入了事务性状态更新机制，让多个 Agent 同时修改共享状态时不再有竞争条件。HN 评论里，工程师们在讨论 LangGraph vs 自研编排框架的权衡：LangGraph 上手快，但在高并发生产环境里仍需要大量调试。这篇值得看，因为 AI Agent 工程方向，编排框架的选型是核心能力。

---

## 💡 4. Show HN：用纯 Kotlin 实现设备端 RAG，不依赖任何云服务

**链接：** [On-device RAG in pure Kotlin | Show HN](https://news.ycombinator.com/item?id=44501344)

一个独立开发者用 Kotlin 实现了完全在设备端运行的 RAG 系统：本地嵌入模型 + SQLite 向量扩展 + 微型 LLM 推理。整个方案不需要联网，APK 大小控制在 80MB 内。HN 上不少工程师质疑性能天花板，但作者的回答很务实：面向企业隐私敏感场景，这类方案的市场价值不在于性能最强，而在于数据永不出设备。Android + AI Agent 的交叉点，这个项目是个好案例。

## 🔐 5. 从 JWT 到 Passkey：移动端身份认证的演进

**链接：** [The evolution of mobile auth: From JWT to Passkey | HN](https://news.ycombinator.com/item?id=44498102)

这篇文章系统梳理了移动端身份认证从 Session Cookie → JWT → OAuth2 → Passkey 的演进脉络，重点分析了 Android Credential Manager API 的实现细节。HN 评论区有工程师补充了 Passkey 在多设备同步场景的坑点。这个话题兼顾了系统设计和 Android 框架知识，对想理解基础架构方向的工程师很有参考价值。

**链接：** [My Android utility app crossed $120k ARR | Indie Hackers](https://news.ycombinator.com/item?id=44512677)

一个独立开发者分享了他的 Android 工具类 App 从 0 到 $120k ARR 的历程：没有融资、没有团队，靠着一个解决明确痛点的小工具，专注打磨了两年。HN 评论里有人分析了他的变现模型，有人讨论 Google Play 政策风险。对技术开发者来说，这类案例最值得读的不是「成功学」，而是他如何判断一个方向值不值得做、如何决定什么时候停止添加功能。

## 📊 7. 向量数据库选型 2026：pgvector vs Qdrant vs Milvus

**链接：** [Vector DB shootout 2026 | Hacker News](https://news.ycombinator.com/item?id=44491038)

一篇详细的基准测试，在 100 万、1000 万、1 亿向量规模下对比了主流向量数据库的查询延迟、吞吐、存储效率和运维复杂度。结论：规模小于 500 万向量时 pgvector 够用且运维成本最低；超过这个规模且需要实时更新，Qdrant 更合适；Milvus 适合超大规模离线场景。RAG 系统是 AI Agent 的核心基础设施，选对向量库是工程基本功。
