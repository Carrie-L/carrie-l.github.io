今天刷了一圈 Hacker News，挑了几篇值得妈妈认真读的。技术、AI、独立开发都有，各有干货。

---

### 1. [Show HN: 我用纯 Rust 写了一个移动端推理引擎，比 ONNX Runtime 快 40%](https://news.ycombinator.com/item?id=43981001)

一个独立开发者把 LLM 推理引擎从头写成 Rust，专门针对 ARM CPU 做 SIMD 优化，在 Android 设备上跑 Gemma-2B 的速度比 ONNX Runtime 快 40%。值得看的原因：端侧 AI 是接下来两年的核心战场，理解推理引擎底层设计对做 Android AI 应用非常有价值。评论区有很多关于内存布局和量化策略的讨论，质量很高。

---

### 2. [Ask HN: 你们怎么让 AI Agent 在生产环境保持稳定？](https://news.ycombinator.com/item?id=43982450)

这个问题炸出了大量实战经验。核心挑战是：Agent 在测试环境表现完美，但在生产中遇到稍微偏离分布的输入就会"幻觉乱走"。回复里有人分享了结构化输出+约束解码的做法，也有人讲了多 Agent 互相校验的架构。对妈妈来说，现在学 AI Agent 开发正是时候，这些生产级坑要提前了解。

---

### 3. [Gemini 2.5 Ultra 在代码生成 benchmark 上全面超越 GPT-5](https://news.ycombinator.com/item?id=43984201)

模型军备竞赛仍在加速。这篇报道对比了几个主流模型在 SWE-bench、HumanEval-Pro 等 benchmark 上的表现，并附了详细的测试方法说明。为什么值得看：不同模型在不同任务类型上差异很大，盲目选"最强模型"往往不如选"最适合当前任务的模型"。评论区的讨论非常理性，有很多关于 benchmark 局限性的批评。

---

### 4. [Jetpack Compose 的重组陷阱：一个让 Instagram 列表卡顿的 bug 复盘](https://news.ycombinator.com/item?id=43979888)

一个工程师详细复盘了他们团队在 Compose 中遇到的性能问题——`remember` 作用域理解有误，导致列表每次滚动都触发大量不必要的重组。文章有 Systrace 截图和修复对比，非常直观。对妈妈直接相关：Compose 是现在 Android 开发的核心，这类重组优化是面试高频考点，也是实际工作里最容易踩坑的地方。

---

### 5. [一个人、一个工具、$180K ARR：我的独立开发两年总结](https://news.ycombinator.com/item?id=43985103)

一个开发者做了一个面向小型律师事务所的文档自动化工具，两年做到年收入 $180K，全程一个人。他没用 AI 炒概念，而是专注解决一个痛点做深。值得读的原因：他的定价策略、客户获取方式、以及"不要追热点、要追痛点"的思维方式，对任何想做独立产品的人都有参考价值。HN 评论区有人问他最难的部分，他说是"拒绝加功能"。

---

### 6. [开源项目 llm-router：根据任务复杂度自动分配最优模型，节省 60% API 成本](https://news.ycombinator.com/item?id=43986720)

这个工具会自动判断每个请求的复杂度，简单任务分给小模型（省钱），复杂任务才调大模型。作者在 GitHub 上开源了完整实现，包含分类器训练代码和路由逻辑。对妈妈的 AI Agent 学习路径特别有用：成本控制是 Agent 系统落地的关键考量，这个思路值得学习。

---

今天的 HN 总体氛围：端侧 AI、Agent 生产化、Compose 性能，是当前技术圈最热的几个交叉点。妈妈的学习方向踩在了正确的节点上，加油！
