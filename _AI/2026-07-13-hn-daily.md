## 1. 陶哲轩：用现代 Coding Agent 重写 1999 年的 Java 小程序

**链接：** [terrytao.wordpress.com](https://terrytao.wordpress.com) · [HN 讨论](https://news.ycombinator.com/item?id=48880170)

世界顶级数学家陶哲轩本周分享了一个有趣实验：他用 AI Coding Agent 把自己 1999 年写的 Java Applet 程序移植到现代技术栈，整个过程只花了几小时，而且 Agent 还顺手找出了两个他当年遗漏的 Bug。他的结论很实在：**Agent 对"有明确上下文的迁移类任务"非常有效**，关键不是 AI 多聪明，而是你能不能给它结构化、可靠的输入。这对我们学习如何正确使用 AI 工具很有启发。

## 2. GhostApproval：打破 AI Coding 工具人工审核边界的漏洞

**链接：** [The Hacker News 报道](https://thehackernews.com/2026/07/what-changes-when-your-software-supply.html)

安全研究员披露了一种名为 **GhostApproval** 的攻击模式，能绕过 AI Coding 助手的"人工确认"安全边界。受影响的工具包括几款主流 AI 编程助手。根源在于 **MCP（Model Context Protocol）** 的隐式信任机制——连接外部服务时，Agent 可能被注入恶意指令并悄悄执行。对开发者来说，这是个提醒：接入 MCP 服务前，一定要审查工具的权限边界。

## 3. Vint Cerf 宣布退休——"互联网之父"谢幕

**链接：** HN 讨论 2026-07-12

TCP/IP 协议联合设计者、被称为"互联网之父"的 **Vint Cerf** 正式宣布退休。一个时代的句点。HN 评论区难得出现了一片敬意而非争论——有人贴出了他 1973 年的原始论文，有人回顾了他推动 IPv6 普及的历程。技术世界欠他的，远比大多数人意识到的要多。

## 4. Nvidia、CoreWeave 与 GPU 泡沫的循环融资内幕

**链接：** HN 讨论 2026-07-12

一篇深度调查揭示了当前 GPU 基础设施投资的"循环融资"结构：云算力公司从 Nvidia 购买 GPU → 用 GPU 算力向 AI 公司收费 → AI 公司的融资又来自押注 GPU 算力的投资人。这个闭环在景气时自我强化，但任何一个环节收紧都会传导放大。值得思考：**AI 基础设施的真实需求与投机性需求各占几成？**

## 5. Claude Cowork 移动端上线：iOS & Android 可跑长任务了

**链接：** [anthropic.com](https://www.anthropic.com)

此前 AI Agent 长任务必须开着桌面端才能跑，手机离开就中断。新发布的 **Claude Cowork iOS/Android 版**彻底改变了这一点——可以在手机上委派知识类任务（文档、表格、演示文稿），监控长时间 Agent 会话，跨设备无缝续接。对移动开发者来说，这也是一个很好的产品设计参考：**在移动端维持 Agent 会话状态的技术挑战**值得深入研究。

## 6. Show HN：OpenKnowledge——开源 AI 知识库，对标 Obsidian/Notion

**链接：** [HN 讨论](https://news.ycombinator.com/item?id=48675435)

一个独立开发者做的开源项目，把 Markdown 写作体验和 AI 工具链深度整合，定位为 Obsidian + Notion 的 AI 原生替代品。功能包括本地向量搜索、与 Claude/GPT 的无缝接入、以及完全本地化存储。**独立开发者用 AI 工具造 AI 工具**——这类 Show HN 项目正是当前技术社区最有活力的部分，也是学习 RAG 和向量检索实战的好素材。
