## 1. Show HN: 我用纯 Rust 写了一个本地 RAG 引擎，无需 Python

**链接**：[https://news.ycombinator.com/item?id=44201837](https://news.ycombinator.com/item?id=44201837)

一位独立开发者分享了用 Rust 实现的本地 RAG（检索增强生成）引擎，不依赖 Python 生态，内存占用极低，可在 4GB RAM 的设备上流畅运行。技术栈包括自实现的向量检索、ONNX 推理运行时以及本地嵌入模型。

**为什么值得看**：RAG 是 AI Agent 的核心组件之一，了解其底层实现能帮助我们在做 Agent 开发时做出更好的架构选择。Rust 实现意味着可以方便地嵌入 Android 应用中运行端侧 AI。

## 7. 论文速读：Agent 的"幻觉工具调用"问题与缓解方案

**链接**：[https://news.ycombinator.com/item?id=44203422](https://news.ycombinator.com/item?id=44203422)

一篇 NeurIPS 2026 预印本，研究 LLM Agent 在调用工具时产生"幻觉参数"的问题（比如传入不存在的文件路径、伪造 API 响应）。作者提出了基于约束解码（Constrained Decoding）的缓解方案，将工具调用准确率提升了 23%。

**为什么值得看**：做 AI Agent 开发必须正视幻觉问题，这不是"偶尔出错"而是系统性风险。了解缓解方案能帮助我们在设计 Agent 时主动加入防护机制，而不是上线后才踩坑。
