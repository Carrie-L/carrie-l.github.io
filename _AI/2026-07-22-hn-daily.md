## 1. Gemma 3n：谷歌面向边缘设备的多模态开源模型

**关键词**：端侧推理、小模型、多模态

谷歌发布 Gemma 3n，专为手机和平板设计，支持图文音频多模态输入，2B/4B 参数可在中低端 Android 设备上本地运行。HN 社区热议：端侧大模型真的够用了吗？联网 API 的时代会提前结束吗？

值得读的原因：端侧 AI 将重塑 Android 应用架构，懂 ML Kit 和 MediaPipe 的 Android 工程师有先天优势，但底层推理框架（ONNX Runtime、LiteRT）才是未来核心竞争力。

## 2. Why I Stopped Using RAG and Started Using Long Context

**关键词**：RAG、长上下文、AI Agent 架构

一篇颇具争议的工程文章：作者用 Gemini 1.5 Pro 的 1M token 上下文替代了整个 RAG 管道，延迟降低、代码减少 70%。HN 顶评反驳：成本在规模化后会爆炸。

值得读的原因：RAG vs Long Context 是当下 AI 工程师必须想清楚的架构抉择，不同场景对应不同解法，没有银弹。

## 3. Kotlin Multiplatform 1.0 正式 Stable 后的一年回顾

值得读的原因：跨平台是大势，KMP 是 Android 开发者进入 iOS 的最低成本路径，早掌握早受益。

## 5. Apple Intelligence 的隐私架构深度解析
