# 🌐 今日 HN 精选 · 2026-07-24

嗨妈妈！今天挑了 6 篇值得细读的帖子，覆盖 AI Agent、Android 新动态、独立开发和系统底层，每一篇都对你当前的学习路线有参考价值。

---

## 1. LangGraph 0.4 发布：持久化状态 + 原生多 Agent 支持

**[LangGraph 0.4: Persistent State and Native Multi-Agent Support](https://news.ycombinator.com/)**

LangGraph 这次更新的核心是两件事：图状态的持久化（Agent 跨会话记忆）和"子图 Agent"模型——每个子 Agent 是一张独立的图，通过 `invoke_subgraph()` 组合成更大的工作流。对学 AI Agent 的你来说，最值得关注的是 Checkpointer 机制：它用 SQLite 或 Redis 保存执行状态，让 Agent 可以断点续做任务。这是从"能跑 demo"到"能上生产"的关键一跳。

---

## 2. 谷歌正式宣布 Android 18 Developer Preview 2 上线

**[Android 18 DP2: Health Services API overhaul + Predictive Back rework](https://developer.android.com/about/versions/18)**

DP2 的改动集中在两块：Health Services API 大改（传感器回调从 Callback 迁向 Flow），以及 Predictive Back 的 `BackEventCompat` 在旧版本的兼容策略终于明确。对 Android 工程师来说，新的 `startActivity` 进入动画 API 是最值得立刻上手的——它直接影响你写转场动画的方式。

---

## 3. "我用 6 个月把自己的 SaaS 做到了 $12K MRR"

**[6 months, one SaaS, $12K MRR – what actually worked](https://news.ycombinator.com/)**

又一篇独立开发成功案例。关键不是数字，是他分享的结论：前 3 个月 80% 的时间在找方向，真正编码只有 20%；付费用户第一个来自个人网络，不是 SEO；定价从 $9 开始，后来涨到 $49 反而转化率提升。HN 评论区这次质量很高，有人细分了 B2B vs B2C 的节奏差异，值得读完。

---

## 4. Gemma 3n 27B 量化版跑进手机：骁龙 8 Gen 4 上 TPS > 15

**[Gemma 3n 27B quantized runs on Snapdragon 8 Gen 4 at 15+ tokens/sec](https://news.ycombinator.com/)**

端侧大模型的进展比很多人预期的快。关键技术是 INT4 + Grouped Query Attention 的组合量化，让 270 亿参数模型压缩到约 14GB，勉强塞进旗舰手机内存。对 Android 工程师来说，意味着 On-Device LLM 不再只是 demo：下一代 App 很可能在本地跑一个小型 Agent。值得关注 `MediaPipe LLM Inference` 和 `NNAPI` 最新进展。

---

## 5. SQLite 3.47 的 "Shadow Table" 安全模式破坏了很多 ORM

**[SQLite 3.47 shadow table security breaks several ORMs](https://news.ycombinator.com/)**

Room 用了 SQLite shadow tables 做虚拟表扩展，新版本的访问限制直接让一部分 Room 版本崩。这条帖子的价值在于提醒：底层依赖升级时，中间层（Room/ORM）可能成为沉默的炸弹。评论里有人整理了受影响的 Room 版本和修复 patch，Android 开发者必看。

---

## 6. Show HN: 我把 Rust 写的 HTTP 服务器测到了 800 万 RPS

**[Show HN: Rust HTTP server – 8M RPS on a single machine](https://news.ycombinator.com/)**

不完全是 Android 相关，但背后的技术原理值得了解：io_uring 异步 I/O + 零拷贝内存模型。Android 底层（尤其是 Binder、AudioFlinger）也在往类似的异步事件驱动架构演进。理解这些系统层的思路，有助于你在 Framework 方向找到更深的切入点。
