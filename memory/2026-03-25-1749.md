### 🔍 为什么最外层包裹 MotionLayout 会导致下拉先展开顶部？
因为 `MotionLayout` 处理嵌套滑动（Nested Scrolling）非常“势利眼”：
它**只认识**直接放在它下面的 `RecyclerView` 或 `NestedScrollView`。一旦你把 RecyclerView 包在了一个自定义的 `NestedScrollableContainer`（本质是 LinearLayout）里面，`MotionLayout` 就变成了“瞎子”！
当你往下拉的时候，`MotionLayout` 根本不知道里面还有个列表没滑到顶，它只觉得：“哦，用户在下拉，那我要优先播放我的展开动画！” —— 于是它毫不客气地把滑动事件**全部抢走**了！

---

### 🗡️ 终极必杀技：回归“大厂黄金架构”！

不要让 `MotionLayout` 当最外层的老大！让它退居二线，**只做头部的动画播放器**！把指挥权交还给 `CoordinatorLayout`！

**这是最完美的布局骨架（只把头部的 UI 塞进 MotionLayout）：**

```xml
<androidx.coordinatorlayout.widget.CoordinatorLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <!-- 1. 真正的滑动大管家 -->
    <com.google.android.material.appbar.AppBarLayout
        android:id="@+id/appBarLayout"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:background="@android:color/transparent"
        app:elevation="0dp">

        <!-- 2. 退居二线的 MotionLayout：只包顶部那点东西！ -->
        <androidx.constraintlayout.motion.widget.MotionLayout
            android:id="@+id/motionLayout"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:minHeight="56dp" <!-- 你的状态栏+标题栏高度 -->
            app:layout_scrollFlags="scroll|exitUntilCollapsed|snap">
            
            <!-- 这里只放头像、背景图、Title，绝对不要放下面的列表！ -->
            
        </androidx.constraintlayout.motion.widget.MotionLayout>
    </com.google.android.material.appbar.AppBarLayout>

    <!-- 3. 下半部分的自定义大容器 -->
    <com.carrie.demoupdatetheme.ui.widget.NestedScrollableContainer
        android:id="@+id/nested_container"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        app:layout_behavior="@string/appbar_scrolling_view_behavior">

        <!-- 你的中间 RV、TabLayout、ViewPager2 全在这里面 -->

    </com.carrie.demoupdatetheme.ui.widget.NestedScrollableContainer>

</androidx.coordinatorlayout.widget.CoordinatorLayout>
```

### 🚨 配合此架构的两步“死命令”：
1. **XML 绞杀令**：去 `MotionScene` 文件里，把 `<OnSwipe ... />` 这一段**彻！底！删！掉！** 绝不让它自己监听手势！
2. **Kotlin 牵线令**：在 Activity/Fragment 里，把 `AppBarLayout` 的滑动偏移量喂给 `MotionLayout`：
   ```kotlin
   appBarLayout.addOnOffsetChangedListener { appBar, offset ->
       val range = motionLayout.height - motionLayout.minimumHeight
       if (range > 0) {
           motionLayout.progress = Math.abs(offset).toFloat() / range
       }
   }
   ```

**🌟 为什么这样就完美了？**
`CoordinatorLayout` 的底层逻辑是极其严谨的：
当你**下拉**时，它会**强制要求内部的列表先滚！** 直到内部的 `RecyclerView` 滚到最顶部 `position 0` 实在滚不动了，才会把剩下的下拉力量交给 `AppBarLayout`，最后带动 `MotionLayout` 展开。完美符合妈妈“先下拉列表，再展开顶部”的需求！
