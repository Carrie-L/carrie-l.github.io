### 🔧 Android 嵌套滑动终极解决方案 (2026-03-25 深夜攻关)
**问题**：MotionLayout 作为最外层包裹所有内容时，下拉会优先展开顶部，而不是先滚动内部列表。
**根因**：MotionLayout 不认识包在自定义 LinearLayout 里的 RecyclerView，导致手势被错误拦截。
**终极解法**：
- 不要让 MotionLayout 做最外层老大
- 使用 CoordinatorLayout + AppBarLayout 作为根布局
- MotionLayout 只包裹顶部区域（Header），放在 AppBarLayout 里
- 下面的 NestedScrollableContainer 设置 `app:layout_behavior="@string/appbar_scrolling_view_behavior"`
- 在代码里用 `addOnOffsetChangedListener` 把 AppBarLayout 的偏移量映射为 MotionLayout 的 progress

### 📚 重要认知更新
- **MotionLayout 嵌套滚动铁律**：MotionLayout 只认识直接放在它下面的 RecyclerView/NestedScrollView。一旦把 RV 包在自定义容器里，它就变成"瞎子"，会错误拦截所有手势。
- **夜间模式状态恢复陷阱**：切换日夜间模式会触发 Configuration Change，导致 Activity 重建。MotionLayout 和自定义容器分开保存/恢复状态会导致坐标错乱。解决方案是在 XML 里给它们加上 `android:saveEnabled="false"`。
