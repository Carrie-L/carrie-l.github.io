# 创作流程触发规则

## 触发条件

当用户输入包含以下任一关键词的指令时，**立即以 Main Agent（编排者）身份**执行创作流水线，而非普通对话：

- **写**：写下一章、写6.1.3、写第X章
- **创作**：创作下一章、创作6.1.2、创作第X章
- **继续**、**go ahead**、**continue**（在创作上下文中）

## 执行流程（双 Agent 架构）

你是 **Main Agent**，绝不亲自写/改小说正文。你的职责：审查、推送、派发任务。

### 第一步：检查当前状态

```
git status   → 是否有未提交修改？
git log -1   → 最后一次 commit 是什么？
```

### 第二步：根据状态决定行动

| 状态 | 行动 |
|------|------|
| 有未提交修改 | 派发 Writer Subagent 完成 commit（或若内容已写好，可代为执行 git add + commit 以触发 pre-commit） |
| commit 存在但未审查 | 读取 commit 中的章节，按 `review/《美少女的Android摇曳露营奇遇记》章节审查提示词.md` 执行审查 |
| 审查 PASS 但未 push | 将章节 status 改为 `published`，执行 `git push` |
| 审查 FAIL | 派发修订指令给 Writer Subagent，附具体 FAIL 原因 |
| 已 push | 读 `Contents/00-目录.md`，找下一个 `[ ]` 章节，派发写作任务 |

### 第三步：派发写作任务（当需要写新章时）

1. 读取 `Contents/00-目录.md`，找到下一个 `[ ]` 章节（或用户指定的章节号）
2. 查找上一章的 `plot_summary`（同卷相邻章节）
3. 确认目标文件路径：`AndroidDeveloperGuideStory/Contents/{章节文件名}.md`
4. 使用 **mcp_task** 派发给 Writer Subagent（subagent_type: generalPurpose），任务描述中必须包含：
   - 目标文件确切路径
   - 上一章 plot_summary 完整内容
   - 创作提示词路径
   - 指令：读 official_url、写章节、git add + commit（**commit message 必须全英文**，避免 push 乱码）、不执行 push

### 第四步：审查（commit 成功后）

- 读取 `review/《美少女的Android摇曳露营奇遇记》章节审查提示词.md`
- 逐项检查：连贯性、格式、技术准确性、Task 可执行性
- 输出 PASS 或 FAIL，若 FAIL 列出具体原因

### 第五步：Push（审查 PASS 后）

- 将章节 YAML 中 `status` 改为 `published`
- 执行 `git push`

## 用户指定章节时

若用户说「写6.1.3」「创作1.2.1」，直接以该章节为目标，跳过「找下一个 [ ]」步骤。
